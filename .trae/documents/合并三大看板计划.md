# 合并三大看板为统一大看板

## 概述

将营收管理看板、供应链管理看板、生产看板合并为一个页面，顶部只保留一个基层单位筛选栏，三个看板内容各为一个分组。

***

## 当前状态分析

### 三个看板的筛选栏对比

| 看板      | 文件                                                                    | 筛选项                                            | 样式                |
| ------- | --------------------------------------------------------------------- | ---------------------------------------------- | ----------------- |
| 营收管理看板  | [RevenueDashboard.vue](src/views/RevenueDashboard.vue) L12-27         | 9个部门(全部/管网/生态/区域/市政/环境建设/管道工程/管道分公司/运营养护/二次养护) | 白色卡片 + 蓝色按钮       |
| 供应链管理看板 | [SupplyChainDashboard.vue](src/views/SupplyChainDashboard.vue) L12-27 | 同上9个                                           | 白色卡片 + 蓝色按钮（完全相同） |
| 生产看板    | [Dashboard.vue](src/views/Dashboard.vue) L11-26                       | 10个(全部+上面9个)                                   | 蓝色渐变头部栏           |

**结论**：营收和供应链的筛选栏**代码完全一致**，生产看板多一个"全部"且样式不同。合并后统一使用一套筛选栏。

### 各看板使用的子组件

| 看板      | 子组件                                                                                                                                                 |
| ------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| 营收管理看板  | **无**（779行全内联）                                                                                                                                      |
| 供应链管理看板 | SupplyChainTimeliness, SupplyContractChart, SupplierStatChart(x2), MaterialProcurementChart                                                         |
| 生产看板    | ProjectStatCard(x5), ManagementTimeliness, RiskTimelinesDisplay, ProductionRiskSituation, ProductionProgress, CertificateProgress, InspectionStatus |

***

### 具体实现步骤

#### 步骤1：创建 UnifiedDashboard.vue

**文件**: 新建 `src/views/UnifiedDashboard.vue`

**模板结构**：

```vue
<template>
  <div class="space-y-6">
    <!-- ====== 统一筛选栏 ====== -->
    <div class="bg-white rounded-xl p-4 shadow-sm">
      <div class="flex flex-wrap gap-2">
        <button v-for="item in filterTabs" :key="item.name"
          :class="[...]" @click="selectedFilter = item.name">
          {{ item.name }} <span>({{ item.count }})</span>
        </button>
      </div>
    </div>

    <!-- ====== 分组一：营收管理看板 ====== -->
    <div class="bg-white rounded-xl shadow-sm overflow-hidden">
      <div class="bg-gradient-to-r from-blue-500 to-blue-600 px-4 py-3">
        <h2 class="text-base font-semibold text-white">营收管理看板</h2>
      </div>
      <!-- 原RevenueDashboard的内容（去掉标题和筛选栏） -->
      ...
    </div>

    <!-- ====== 分组二：供应链管理看板 ====== -->
    <div>
      <div class="bg-white rounded-xl p-4 shadow-sm mb-2">
        <div class="flex items-center gap-2">
          <svg .../>  <h2 class="text-lg font-semibold">供应链管理看板</h2>
        </div>
      </div>
      <!-- 原SupplyChainDashboard的grid布局+4个子组件 -->
      ...
    </div>

    <!-- ====== 分组三：生产看板 ====== -->
    <div>
      <div class="bg-gradient-to-r from-blue-600 to-blue-400 rounded-lg p-3 shadow-sm">
        <h2 class="text-lg font-bold text-white">生产看板</h2>
        <div class="text-right">项目总数 {{ totalProjectCount }}</div>
      </div>
      <!-- 原Dashboard.vue的grid布局+7个子组件 -->
      ...
    </div>
  </div>
</template>
```

**Script**：

* 合并三个看板的 import（所有子组件 + mockData）

* 统一的 `selectedFilter` ref 和 `filterTabs`

* 将 `selectedFilter` 作为 prop 传递给各子组件

* 保留各看板原有的事件处理（drill-down, navigate等）

#### 步骤2：修改 App.vue 的 viewMap

**文件**: [App.vue](src/App.vue)

* 导入新组件：`import UnifiedDashboard from './views/UnifiedDashboard.vue'`

* 在 viewMap 中添加映射（或替换原有三个）

* **保留**原有的三个独立入口作为备用，同时新增一个"综合看板"菜单项

具体修改：

```js
// 新增导入
import UnifiedDashboard from './views/UnifiedDashboard.vue'

// viewMap 新增
const viewMap = {
  // ... 原有保持不变
  '综合看板': markRaw(UnifiedDashboard),   // 新增
}
```

同时在 Sidebar 中为"产运管理"下新增"综合看板"菜单项。

#### 步骤3：处理各子组件的 currentUnit prop

各子组件已接受 `currentUnit` prop，UnifiedDashboard 将统一的 `selectedFilter` 传递下去即可。
