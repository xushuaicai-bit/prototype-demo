# 供应链看板下钻筛选修复计划

## 概述

修复供应链管理看板中各组件（及时性卡片、合同图表、分类区域）点击下钻至「供应商合同签订报表」时，数据未能按对应状态（待签订/已签订/红橙黄预警）和分类正确筛选的问题。

## 当前状态分析

### 完整下钻链路（经代码验证）

```
用户点击 → 子组件 emit('drill-down', filter)
  → SupplyChainDashboard.handleDrillDown() 合并 unit
    → emit('navigate', {view:'supply-chain', report:'supplier-contract', filter})
      → App.handleNavigate() 保存 navigateFilter + 切换视图
        → SupplyChainReport 传递 reportFilter 给 SupplierContractReport
          → SupplierContractReport.applyInitialFilter() 映射到各 select ref
            → tableData computed 过滤渲染
```

### 定位的 Bug

**Bug 1（主要问题）：SupplyContractChart 全部模式（堆叠柱状图）点击缺少状态筛选**

当用户选择"全部"查看堆叠柱状图时，点击任意颜色段（未签/黄色预警/橙色预警/红色预警），当前代码只传了 `{ unit: 单位名 }`，**没有传递签约状态或超期天数的筛选条件**。导致下钻后显示该单位的全部合同而非点击的那个状态分段的数据。

**Bug 2（需排查）：可能存在的 filter 应用时序问题**

需要确认在从看板切换到报表时，`initialFilter` prop 的值是否在组件挂载时已经就绪。

## 修改方案

### 修改文件 1：`src/components/dashboard/SupplyContractChart.vue`

**修改位置**：第 291-310 行，chart.on('click') 回调函数

**修改内容**：在"全部模式"分支中，增加对点击数据系列名称（seriesName）的判断，根据点击的具体颜色段补充对应的筛选条件：

```javascript
// 修改前（全部模式）：
} else {
  if (params.name && params.name !== '全部') {
    filter.unit = params.name
  }
}

// 修改后（全部模式）：
} else {
  if (params.name && params.name !== '全部') {
    filter.unit = params.name
    // 根据点击的堆叠段（seriesName）补充状态筛选
    if (params.seriesName === '未签') {
      filter.signStatus = '未签订'
    } else if (params.seriesName === '红色预警') {
      filter.signStatus = '未签订'
      filter.overdueDays = '60+'
    } else if (params.seriesName === '橙色预警') {
      filter.signStatus = '未签订'
      filter.overdueDays = '45-60'
    } else if (params.seriesName === '黄色预警') {
      filter.signStatus = '未签订'
      filter.overdueDays = '21-44'
    }
  }
}
```

### 修改文件 2：`src/components/supply-chain/SupplierContractReport.vue`（可选加固）

**修改位置**：`applyInitialFilter` 函数及 `watch` 逻辑

**目的**：确保 filter 应用更可靠，添加 nextTick 防止时序问题，并增加日志便于调试。
