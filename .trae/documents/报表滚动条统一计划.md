# 报表左右滚动条统一计划

## 目标
将所有 el-table 报表的横向滚动条统一为**表内滚动**（el-table 内部原生横向滚动）：滚动条统一出现在表格底部内侧，左侧 fixed 列（基层单位/序号/项目编号/项目名称）滚动时固定可见，合计行（show-summary）跟随表格横向滚动。

## 现状分析（Phase 1 实际读取确认）

### 当前滚动方式混杂

| 报表文件 | 外层 overflow-x-auto | el-table min-width style | 合计行 | 实际滚动 |
|---|---|---|---|---|
| RevenueDetail.vue (L130) | 有 | 无 | show-summary | 表内 |
| RevenueSummary 施工 (L118/L123) | 有 | 3500px | 原生 table(sticky) | 表外 |
| RevenueSummary 产品 (L510/L517) | 有 | 800px | 无 | 表外 |
| RevenueSummary 其他 (L617/L624) | 有 | 600px | 无 | 表外 |
| RevenueDeviation 当月 (L97/L103) | 有 | 1800px | show-summary | 表外 |
| RevenueDeviation 季度 (L453/L458) | 有 | 2200px | 无 | 表外 |
| fund 4个 (ProjectStatusSummary等) | 无 | 无 | 部分 | 表内 |
| market el-table 4个 + MarketTracking + BidProject | 有 | 无 | 无 | 表内(外层不触发) |
| supply-chain 2个 (SupplierDetail/Contract) | 有 | 无 | 无 | 表内(外层不触发) |
| BidSummaryReport.vue (L22) | 有 | 原生table min-width:1200px | - | 表外(原生table) |

### 问题
- RevenueSummary/RevenueDeviation 设了 el-table min-width → 触发外层 div 滚动（表外），同时外层 overflow-x-auto 与 el-table 内部滚动潜力并存 → 部分报表"表里表外都有"
- 表外滚动导致 fixed 列失效（基层单位/序号等被带走）
- 滚动条位置不统一（有的在表内底部，有的在表外容器底部）

### RevenueSummary 施工类特殊结构
- L118 外层 overflow-x-auto 包裹：el-table(L123, min-width:3500px) + 原生 `<table>` 合计行(L463, min-width:3500px, sticky) + el-pagination
- 原生 table 合计行与 el-table 分离，靠外层 overflow-x-auto 同步滚动
- 改表内滚动后，原生 table 无法跟随 el-table 内部滚动 → 必须改为 el-table 原生 show-summary
- CSS L1380-1389 `.total-footer-row`（仅施工类用），L1391-1399 `:deep(.el-table__footer-wrapper)` sticky（保留，让所有 el-table 合计行 sticky 底部）

## 改造方案

### 核心原则
1. 移除所有 el-table 报表外层 `<div>` 的 `overflow-x-auto` class（保留 div 本身与其它属性，避免破坏布局）
2. 移除 el-table 上的 `style="...min-width:..."`（让 el-table 默认 width:100%，列总宽超出时表内横向滚动，fixed 列生效）
3. RevenueSummary 施工类原生 table 合计行 → el-table show-summary
4. BidSummaryReport（原生 `<table>`，非 el-table）保留 overflow-x-auto 作为例外（原生 table 无表内滚动机制）

### 文件级改动

#### 1. RevenueSummary.vue（核心改造）
- **L118**：`<div v-if="activeTab === 'construction'" class="overflow-x-auto">` → `<div v-if="activeTab === 'construction'">`
- **L123**：el-table 移除 `style="min-width: 3500px;"`；新增 `show-summary` 和 `:summary-method="getConstructionSummaries"` 和 `:summary-cell-style="{ background: '#f9fafb', fontWeight: 'bold' }"`
- **L461-496**：删除整个 `<!-- 固定底部合计行 -->` div（原生 table 合计行）
- **L510**：`<div v-if="activeTab === 'product'" class="overflow-x-auto">` → `<div v-if="activeTab === 'product'">`
- **L517**：el-table 移除 `style="width: 100%; min-width: 800px;"`
- **L617**：`<div v-if="activeTab === 'other'" class="overflow-x-auto">` → `<div v-if="activeTab === 'other'">`
- **L624**：el-table 移除 `style="width: 100%; min-width: 600px;"`
- **script**：新增 `getConstructionSummaries` 函数（基于 constructionTotals 返回合计数组，第一列返回"合计"，completionRate 返回百分比，其余返回 formatNumber）
- **style L1380-1389**：删除 `.total-footer-row` 和 `.total-footer-row td` CSS（不再使用）；保留 `:deep(.el-table__footer-wrapper)` sticky

新增函数：
```js
const getConstructionSummaries = ({ columns }) => {
  return columns.map(col => {
    const totals = constructionTotals.value || {}
    if (col.property === 'name') return '合计'
    if (col.property === 'completionRate') {
      return totals.completionRate != null ? totals.completionRate.toFixed(2) + '%' : ''
    }
    const val = totals[col.property]
    return val != null ? formatNumber(val) : ''
  })
}
```
