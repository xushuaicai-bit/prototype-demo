# 营收看板UI微调计划

## 一、需求概述

对 `RevenueDashboard.vue` 进行5项UI调整：

1. 施工类营收情况 + 总体营收情况 改为并排（一行两列）
2. 营收计划上报及时性：从3个圆环精简为1个圆环
3. 当年累计完成营收情况：底部统计卡片改为标题下一行文字描述
4. 当年累计完成营收情况tooltip：每个柱子显示全部区域项目数量汇总
5. 月度营收偏差：仅保留年月选择器+第1个偏差模块

### 改动2: 营收计划上报及时性 — 精简为单圆环

### 改动4: 当年累计完成营收情况 tooltip — 显示全部区域汇总

**文件**: \[RevenueDashboard.vue] 第657-692行（initSectorChart内的regionMarketMap和formatter）

**当前行为**: 每个柱子悬浮时只显示该业务类型对应的单个区域（如"水务→上海"）

**改为**: 每个柱子的tooltip都展示所有6个区域的项目数量汇总表：

```
┌──────────────────────────┐
│ 水务                     │
│ ● 累计营收: 32000万元    │
│ ● 项目数量: 15个         │
│ ──────────────────────── │
│ 区域分布:               │
│ 上海: 25个              │
│ 长三角总部: 24个        │
│ 中原总部: 16个          │
│ 大湾区总部: 18个        │
│ 境外区域: 3个           │
│ 其他城市区域: 10个      │
│ 合计: 63个项目          │
└──────────────────────────┘
```

**实现方式**:

* 定义全局区域汇总数据（不按业务类型区分）：

```javascript
const allRegionSummary = [
  { name: '上海', count: 25 },
  { name: '长三角总部', count: 24 },
  { name: '中原总部', count: 16 },
  { name: '大湾区总部', count: 18 },
  { name: '境外区域', count: 3 },
  { name: '其他城市区域', count: 10 }
]
```

* tooltip formatter中遍历 allRegionSummary 输出所有区域

* 删除 regionMarketMap 按名称查询单区域的逻辑
