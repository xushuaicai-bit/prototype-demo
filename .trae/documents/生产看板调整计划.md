# 生产看板调整计划

## 概述

对生产看板进行2大块调整：

1. **管理提示及时性组件**布局重排：调换预警位置、重组指标行
2. **风险预警及时性组件**UI优化：按内置数量关系分组显示

## 详细修改方案

### 调整1：管理提示及时性布局重排

#### 1.1 调换预警位置 + 置于首行

**文件**: [ManagementTimeliness.vue](src/components/dashboard/ManagementTimeliness.vue)

将第1行的两个预警卡片**左右调换**：

* 左侧：**筹划预警**（原右侧，bg-orange-50）

* 右侧：**登记预警**（原左侧，bg-red-50）

#### 1.2 第二行：待登记/已登记/应登记（可点击下钻）

当前数据中只有 `应登记`(508) 和 `待登记`(300)，缺少 `已登记`。

**方案**：在 mockData 的 subItems 中新增 `已登记` 字段（或在前端计算），值 = 应登记 - 待登记 = 508 - 300 = 208

第二行展示顺序：**待登记 | 已登记 | 应登记**，整体可点击，点击后 emit `drill-to-pending`

**mockData.js subItems 新顺序**：

```js
subItems: [
  { name: '待登记', value: 300 },          // index 0 → 第2行第1个
  { name: '已登记', value: 208 },           // index 1 → 第2行第2个 (新增，=应登记-待登记)
  { name: '应登记', value: 508 },           // index 2 → 第2行第3个
  { name: '应筹划', value: 508 },           // index 3 → 第3行第1个
  { name: '待筹划', value: 300 },           // index 4 → 第3行第2个
  { name: '项目筹划完成率', value: '45%' }, // index 5 → 第3行第3个
  { name: '基本信息完成率', value: '72%' }  // index 6 → 第4行单独
]
```

#### 1.3 第三行：应筹划 / 待筹划 / 项目筹划完成率

使用 subItems\[3]、subItems\[4]、subItems\[5]，grid-cols-3，不可点击。

#### 1.4 第四行：基本信息完成率（单独一行）

使用 subItems\[6]，保持现有渐变背景样式不变。

#### 调整后的完整布局

```
第1行： [筹划预警(左)]  |  [登记预警(右)]        ← 预警位置互换，仍为首行
第2行： [待登记] [已登记] [应登记]               ← 整体可点击下钻至待登记
第3行： [应筹划] [待筹划] [项目筹划完成率]        ← 三个放一排
第4行： [基本信息完成率 渐变高亮]                ← 单独一行
```

### 调整2：风险预警及时性 UI 分组优化

#### 内置数量关系（用户定义）

```
组A：当年一、二级风险数量 ∈ 当前进入风险数量
      ┌─────────────┐  ┌──────────────┐
      │ 当年一、二级 │  │ 当前进入风险 │
      │ 风险数量    │  │ 数量         │
      └─────────────┘  └──────────────┘

组B：当前进入风险数量 = 当年完成风险数量 + 当年剩余风险数量（打包一组）
      ┌─────────────────────────────────────┐
      │  当前进入风险数量                    │
      │  ┌──────────┐ ┌──────────┐          │
      │  │当年完成  │ │当年剩余  │          │
      │  │风险数量  │ │风险数量  │          │
      │  └──────────┘ └──────────┘          │
      └─────────────────────────────────────┘

组C：未来两周进入风险数量 = I级 + II级 + III级（打包一组）
      ┌─────────────────────────────────────────────┐
      │  未来两周进入风险数量                         │
      │  ┌────┐ ┌─────┐ ┌──────┐                   │
      │  │I级 │ │II级 │ │III级 │                   │
      │  └────┘ └─────┘ └──────┘                   │
      └─────────────────────────────────────────────┘
```

#### UI 实现方案

**文件**: [RiskTimelinessDisplay.vue](src/components/dashboard/RiskTimelinessDisplay.vue)

将当前的扁平 `grid-cols-2` 列表改为**分层分组卡片**布局：

**结构设计**：

```
┌─────────────────────────────────────────┐
│ 风险预警及时性                           │
│                                         │
│ ┌──────────┐ ┌────────────────────────┐ │
│ │当年一、二 │ │ 当前进入风险数量  2个   │ │
│ │级风险数量│ │ ┌──────┐ ┌──────┐     │ │
│ │   2个    │ │ │当年完 │ │当年剩 │     │ │
│ └──────────┘ │ │成数量 │ │余数量 │     │ │
│              │ │ 2个   │ │ 2个   │     │ │
│              │ └──────┘ └──────┘     │ │
│              └────────────────────────┘ │
│                                         │
│ ┌─────────────────────────────────────┐ │
│ │ 未来两周进入风险数量（按等级+总量）2个 │ │
│ │ ┌────┐ ┌─────┐ ┌──────┐           │ │
│ │ I级 │ │ II级│ │ III级│            │ │
│ │ 2个 │ │ 2个 │ │ 2个  │            │ │
│ └────┘ └─────┘ └──────┘             │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

具体实现：

* **第一行**：左边小卡 `当年一、二级风险数量` + 右边大卡片 `当前进入风险数量`（内含子项：当年完成 + 当年剩余）

* **第二行**：全宽卡片 `未来两周进入风险数量`（内含子项：I级 + II级 + III级）

样式要点：

* 组B（当前进入）用浅蓝色/蓝色边框区分

* 组C（未来两周）用浅橙色/橙色边框区分

* 组A（一、二级）与组B同行但独立卡片

* 子项缩进显示层级关系
