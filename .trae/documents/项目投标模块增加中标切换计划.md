# 项目投标模块增加中标切换计划

## 概述
在市场管理看板的"项目投标（数量+金额）"区块增加大模块切换功能，支持在"投标"和"中标"两个模块间切换，默认显示"投标"。切换后标题、预警指标、图表数据及图例名称均相应变化。

## 当前状态分析

### 项目投标区块 (MarketDashboard.vue L117-151)
- **预警指标框**：显示"应开标数量"，值 `shouldOpenBidCount`（12），可点击下钻 `drillDownToBid`
- **标题行**：左侧标题"项目投标（数量+金额）"，右侧子标签切换（`bidTabs`：分公司/按区域/按项目类型）
- **图表**：`bidChartRef`，由 `updateBidChart()` 渲染，图例为"A级投标/AA级投标/AAA级投标/投标金额"

### 图表管理逻辑
- `activeBidTab` ref 控制子标签（company/region/type）
- `updateBidChart()` 根据 `activeBidTab` 选择不同数据，渲染到 `charts[2]`
- `initDualAxisChart()` 为通用双轴图表初始化函数
- `getCategories(tabKey)` 返回分类轴数据

## 修改方案

### 1. 模板修改 (MarketDashboard.vue template)

在预警指标框和标题行之间，增加大模块切换按钮组：
```html
<!-- 大模块切换 -->
<div class="flex space-x-2 mb-3">
  <button
    v-for="mode in bidModules"
    :key="mode.key"
    :class="[
      'px-4 py-1.5 rounded-lg text-sm font-medium transition-all',
      activeBidModule === mode.key
        ? 'bg-blue-600 text-white shadow-md'
        : 'bg-gray-100 text-gray-600 hover:bg-blue-50 hover:text-blue-600'
    ]"
    @click="switchBidModule(mode.key)"
  >
    {{ mode.label }}
  </button>
</div>
```

动态标题：
```html
<h3 class="text-base font-semibold text-gray-800">项目{{ activeBidModule === 'bid' ? '投标' : '中标' }}（数量+金额）</h3>
```

预警指标动态化 — 投标模式显示"应开标数量"，中标模式显示"已中标数量"：
```html
<span class="text-sm font-medium text-red-700">{{ activeBidModule === 'bid' ? '应开标数量' : '已中标数量' }}</span>
<span class="text-lg font-bold text-red-600">{{ activeBidModule === 'bid' ? shouldOpenBidCount : wonBidCount }}</span>
```

### 2. Script 修改 (MarketDashboard.vue script)

**2a. 新增状态：**
```js
const activeBidModule = ref('bid') // 'bid' | 'winning'
const wonBidCount = ref(18) // 中标模式预警指标值
```

**2b. 新增大模块配置：**
```js
const bidModules = [
  { key: 'bid', label: '投标' },
  { key: 'winning', label: '中标' }
]
```

**2c. 新增切换函数：**
```js
const switchBidModule = (key) => {
  activeBidModule.value = key
  updateBidChart()
}
```

**2d. 修改 `updateBidChart()` 函数：**
- 根据 `activeBidModule` 选择不同数据集
- 投标模式：使用现有投标数据
- 中标模式：使用新增中标数据（数值小于投标，符合实际业务逻辑）
- 图例和系列名称动态化：`'投标'` ↔ `'中标'`

中标数据（Mock，各子标签三组）：
- company（9个分公司）：数量约为投标的60-70%
- region（6个区域）：数量约为投标的60-70%
- type（8个类型）：数量约为投标的60-70%

修改后的 `updateBidChart()` 结构：
```js
const updateBidChart = () => {
  const categories = getCategories(activeBidTab.value)
  const isWinning = activeBidModule.value === 'winning'
  const label = isWinning ? '中标' : '投标'

  let aData, aaData, aaaData, amountData
  if (isWinning) {
    // 中标数据
    if (activeBidTab.value === 'region') { ... }
    else if (activeBidTab.value === 'type') { ... }
    else { ... }
  } else {
    // 现有投标数据（保持不变）
    ...
  }

  const chartData = {
    legend: [`A级${label}`, `AA级${label}`, `AAA级${label}`, `${label}金额`],
    categories: categories,
    series: [
      { name: `A级${label}`, data: aData, yAxisIndex: 0, color: '#3b82f6' },
      { name: `AA级${label}`, data: aaData, yAxisIndex: 0, color: '#10b981' },
      { name: `AAA级${label}`, data: aaaData, yAxisIndex: 0, color: '#f59e0b' },
      { name: `${label}金额`, data: amountData, yAxisIndex: 1, color: '#8b5cf6' }
    ]
  }
  // 图表渲染逻辑不变
}
```

### 3. 不需要修改的部分
- `initDualAxisChart()` 函数 — 通用函数，无需修改
- `charts` 数组管理 — 仍使用 `charts[2]`
- 子标签切换逻辑 — `activeBidTab` 和 `bidTabs` 保持不变
- `initCharts()` — 初始调用 `updateBidChart()` 自动使用默认 'bid' 模式

## 假设与决策
1. **UI 位置**：大模块切换按钮放在预警指标框下方、标题行上方，使用稍大的按钮样式区分于子标签。
2. **中标数据**：使用 Mock 数据，数值约为投标数据的60-70%（符合中标率业务逻辑）。
3. **预警指标**：投标模式显示"应开标数量"（12），中标模式显示"已中标数量"（18）。
4. **下钻链接**：暂不修改，两种模式共用现有 `drillDownToBid` 下钻链接。
5. **图表复用**：两种模式共用 `bidChartRef` 和 `charts[2]`，切换时 dispose 重建。
