# 供应商合同签订报表 — 未签订合同字段置空计划

## Summary

## Proposed Changes

### 改动方案：为每个目标列添加 `#default` template，判断未签订时显示空

涉及 L248-L337 共 **12 列**，其中：
- **无 template 的列**（8列）：合同分类、供应商名称、分包合同名称、工作范围及内容、主要工程量、合同选用形式、计划签订日期、实际签订日期、合同开始时间、合同计划/实际结束时间 — 需新增 `#default` template
- **已有 template 的列**（2列）：供应商等级、初始策划金额 — 需修改现有 template

**判断条件**：`scope.row.signStatus === '未签订'`

**修改模式**：

对无 template 的列，从：
```html
<el-table-column label="合同分类" prop="category" width="130" align="center" />
```
改为：
```html
<el-table-column label="合同分类" prop="category" width="130" align="center">
  <template #default="scope">
    {{ scope.row.signStatus === '未签订' ? '' : scope.row.category }}
  </template>
</el-table-column>
```

对已有 template 的列（如供应商等级），在现有逻辑前加判断：
```html
<template #default="scope">
  <span v-if="scope.row.signStatus === '未签订'"></span>
  <span v-else :class="getSupplierLevelClass(scope.row.supplierLevel)">
    {{ scope.row.supplierLevel }}
  </span>
</template>
```

对初始策划金额列同理：
```html
<template #default="scope">
  {{ scope.row.signStatus === '未签订' ? '' : formatNumber(scope.row.plannedAmount) }}
</template>
```

## Assumptions & Decisions

1. 判断条件使用 `signStatus === '未签订'`，与之前新增的 `getDisplayContractStatus` 逻辑一致
2. 置空范围不包含"超期天数"列（该列不在指定范围内）
3. 置空范围不包含"项目编号"到"项目状态"这几列（这些是项目级信息，非合同级）
4. 不修改 rawData 数据本身，仅控制显示

## Verification Steps

1. 打开供应商合同签订报表 → 未签订的合同行中，"合同分类"到"合同计划/实际结束时间"之间的所有列显示为空
2. 已签订的合同行各字段值正常显示，不受影响
3. 运行 GetDiagnostics 确认无语法错误
