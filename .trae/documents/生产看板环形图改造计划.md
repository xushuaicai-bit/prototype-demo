# 生产看板：管理提示及时性 + 风险预警及时性 改为环形图

## 概述
将生产看板中的"管理提示及时性"和"风险预警及时性"两个组件，从当前的卡片/指标列表布局改为**ECharts 环形图（Donut Chart）**展示。

## 当前状态分析

### ManagementTimeliness.vue（当前布局）
- 第1行：筹划预警(左) + 登记预警(右)，各含红/橙/黄三色数值
- 第2行：待登记 | 已登记 | 应登记（可点击下钻）
- 第3行：应筹划 | 待筹划 | 项目筹划完成率
- 第4行：基本信息完成率（渐变高亮）
- 数据源：`managementTimelinessData`（含 registrationAlerts、planningAlerts、subItems 7项）

### RiskTimelinessDisplay.vue（当前布局）
- 分组卡片：当年一、二级 + 当前进入风险(含子项)
- 分组卡片：未来两周进入风险(含I/II/III级子项)
- 数据源：`riskAlertTimeliness`（8项数组）

### 项目中已有的环形图参考
- [SupplyContractChart.vue](src/components/dashboard/SupplyContractChart.vue#L109-L143)：ECharts 饼图模式 `radius: ['40%', '70%']`
- [ProductionDashboard.vue](src/views/ProductionDashboard.vue#L262-L278)：巡检计划饼图
- [SupplyChainTimeliness.vue](src/components/dashboard/SupplyChainTimeliness.vue#L94-L120)：SVG 圆形进度环

## 详细修改方案

### 调整1：ManagementTimeliness.vue → 双环形图

**设计思路**：用两个并排的环形图分别展示**登记**和**筹划**两大维度的完成情况。

#### 左侧环形图 — 合同登记情况
- 数据来源：
  - 已登记 = subItems[1].value = 208
  - 待登记 = subItems[0].value = 300
  - 总计 = subItems[2].value = 508
- 环形图数据（pie series）：
  - `{ name: '已登记', value: 208, itemStyle: { color: '#22c55e' } }`
  - `{ name: '待登记', value: 300, itemStyle: { color: '#e5e7eb' } }`
- 中心文字：显示完成率 `41%` + "已登记/应登记"
- 下方附加：登记预警红/橙/黄小标签

#### 右侧环形图 — 项目筹划情况
- 数据来源：
  - 待筹划 = subItems[4].value = 300
  - 应筹划 = subItems[3].value = 508
  - 完成数 = 508 - 300 = 208
- 环形图数据：
  - `{ name: '已完成筹划', value: 208, itemStyle: { color: '#3b82f6' } }`
  - `{ name: '待筹划', value: 300, itemStyle: { color: '#e5e7eb' } }`
- 中心文字：显示完成率 `41%` + "已完成/应筹划"
- 下方附加：筹划预警红/橙/黄小标签

#### 底部一行 — 两个完成率指标
- 左侧：项目筹划完成率 45%（小卡片或文字）
- 右侧：基本信息完成率 72%（渐变高亮卡片）

#### 技术实现
- 使用 ECharts `type: 'pie'`, `radius: ['45%', '70%']` 环形图
- 两个图表左右排列（grid-cols-2），各占一个 `<div ref>` 
- 中心文字通过 `label` + `graphic` 或自定义 formatter 实现
- 保持 `drill-to-pending` emit 事件（点击环形图触发下钻）

### 调整2：RiskTimelinessDisplay.vue → 单环形图 + 子项说明

**设计思路**：用一个主环形图展示风险等级分布，下方保留关键指标的分组说明。

#### 主环形图 — 风险等级分布
- 数据来源：`riskByLevel` 或从 `riskAlertTimeliness` 推导
- 环形图数据：
  - `{ name: 'I级风险', value: 2, itemStyle: { color: '#ef4444' } }`
  - `{ name: 'II级风险', value: 6, itemStyle: { color: '#f97316' } }`
  - `{ name: 'III级风险', value: 8, itemStyle: { color: '#eab308' } }`
- 中心文字："当前进入风险数量" + "2个"

#### 下方 — 分组指标说明（紧凑版）
保持现有的分组逻辑但更紧凑：
- 第一行小标签：当年一、二级(2个) | 当年完成(2个) | 当年剩余(2个)
- 第二行小标签：I级两周(2个) | II级两周(2个) | III级两周(2个)

#### 技术实现
- 上半部分：ECharts 环形图（占约60%高度）
- 下半部分：紧凑的指标网格（占约40%高度）
- 引入 echarts 并使用 ref 初始化图表

## 假设与决策

1. **图表库**：使用项目中已集成的 ECharts（与其他组件一致）
2. **环形图样式**：`radius: ['45%', '70%']`，带圆角边框 `borderRadius: 10`
3. **中心文字**：通过 ECharts `graphic` 元素或 `title` + 自定义 label 实现
4. **交互**：环形图支持 tooltip 和 click 事件，保持现有下钻能力
5. **响应式**：图表在窗口 resize 时自适应
