# 销项项目汇总表 — 分类字段修复与图表删除计划

## 概述
修改「资金管理 → 销项项目汇总」（`OutputProjectSummary.vue`）：更新一/二/三级分类的选项值与映射关系，将第一列"分公司"改名为"基层单位"，删除"各分公司合同价与审定价对比"可视化图表，并确保一级分类列正常显示。

## 现状分析
- 目标组件：[OutputProjectSummary.vue](file:///e:/trae_demo_env/营收看板/src/components/fund/OutputProjectSummary.vue)
- 第 167-168 行：前两列 `分公司`(prop=branch) 和 `一级分类`(prop=category1) 均设 `fixed="left"`
- 第 251 行：`category1Options` 有 8 个值（水务/水环境治理/水利/城市更新/市政路桥房建/固废处理与处置/土壤修复/其他），用户只需 4 个
- 第 254-263 行：`category2Map` 的二级分类值与用户指定值（城镇水务/水处理/清洁小流域治理/水工构筑物工程/旧城改造/城市空间提升等）不一致
- 第 266-292 行：`category3Map` 的三级分类值与用户指定值（供水/排水/生活污水处理等）不一致
- 第 127-149 行：模板中"各分公司合同价与审定价对比"图表区（需删除）
- 第 762-788 行：`branchChartData` computed（图表数据，需删除）
- 第 1005-1097 行：图表相关 CSS（需删除）
- 第 24 行：筛选区"分公司："标签（需改名为"基层单位："）
- 第 904 行：导出表头 '分公司'（需改名为 '基层单位'）
- 树形结构：基层单位 → 一级分类 → 二级分类 → 三级分类，每层节点仅填充自己列的值

## 改动方案（单文件：`src/components/fund/OutputProjectSummary.vue`）

### 1. 第一列"分公司"改名为"基层单位"（仅改显示文本，prop 保持 branch）
- 第 167 行：`label="分公司"` → `label="基层单位"`
- 第 24 行：筛选标签 `label="分公司："` → `label="基层单位："`
- 第 904 行：导出表头 `'分公司'` → `'基层单位'`
- 内部变量名 `branchOptions`、`filters.branches`、prop `branch` 保持不变（仅显示文本改变）

### 2. 更新一级分类选项
- 第 251 行：`category1Options` 改为：
  ```js
  const category1Options = ['水务', '水环境治理', '城市更新', '其他']
  ```

### 3. 更新二级分类映射（category2Map）
- 第 254-263 行：替换为：
  ```js
  const category2Map = {
    '水务': ['城镇水务', '水处理'],
    '水环境治理': ['清洁小流域治理', '水工构筑物工程'],
    '城市更新': ['旧城改造', '城市空间提升'],
    '其他': ['其他工程']
  }
  ```

### 4. 更新三级分类映射（category3Map）
- 第 266-292 行：替换为：
  ```js
  const category3Map = {
    '城镇水务': ['供水', '排水'],
    '水处理': ['生活污水处理', '再生水利用'],
    '清洁小流域治理': ['河道清淤', '生态护岸'],
    '水工构筑物工程': ['泵站', '水闸'],
    '旧城改造': ['房屋征收', '基础设施改造'],
    '城市空间提升': ['景观改造', '功能提升'],
    '其他工程': ['其他']
  }
  ```

### 5. 删除"各分公司合同价与审定价对比"图表
- **模板**：删除第 127-149 行整个 `<div class="chart-section">...</div>` 块
- **脚本**：删除第 762-788 行 `branchChartData` computed 及其上方的注释 `// ============================ 图表数据 ============================`
- **CSS**：删除第 1005-1097 行所有图表相关样式（.chart-section / .chart-title / .bar-chart / .bar-group / .bar-label / .bar-row / .bar / .bar-contract / .bar-audit / .bar-text / .chart-legend / .legend-item / .legend-color / .legend-color.contract / .legend-color.audit）

### 6. 确保一级分类列正常显示
- 当前第 168 行 `category1` 列有 `fixed="left"`，与第 167 行 `branch` 列同为 fixed left。两个 fixed-left 列在树形表中可能导致第二列渲染异常。
- **修复**：移除 `category1` 列的 `fixed="left"`（仅保留 `branch` 列 fixed left），确保一级分类列内容正常显示。即第 168 行改为：
  ```html
  <el-table-column prop="category1" label="一级分类" width="120" />
  ```
- 实施后需在浏览器中验证一级分类列的值（水务/水环境治理/城市更新/其他）在树形展开后正常显示。

## 假设与决策
1. **基层单位**：仅改列名显示文本，prop/变量名/数据值（管道工程/环境建设/运营养护/区域发展）不变。用户已确认"仅改列名"。
2. **分类映射**：用户已认可提议的 一级→二级→三级 映射方案。
3. **"等"的处理**：用户给出的二/三级分类值带"等"字，按提议的具体值实现，后续可增补。
4. **不新增文件**：所有改动集中在 `OutputProjectSummary.vue`。
5. **图表删除**：模板、computed、CSS 三处一并清除，不留无用代码。
