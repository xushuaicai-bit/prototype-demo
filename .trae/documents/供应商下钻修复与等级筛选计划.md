## 修改步骤

### 步骤1：修复"未签订"下钻匹配逻辑（核心修复）
**文件**: [SupplierContractReport.vue](src/components/supply-chain/SupplierContractReport.vue#L1156-L1158)
**问题**: 当 `selectedSignStatus === '未签订'` 时，当前用精确匹配 `getWarningLabel(contract) !== '未签订'` 会过滤掉红/橙/黄预警的合同
**改为**: "未签订" 应匹配所有非'已签订'的合同

```js
// 当前代码 (L1156-1158):
if (selectedSignStatus.value) {
  const statusVal = contract.signStatus === '已签订' ? '已签订' : getWarningLabel(contract)
  if (statusVal !== selectedSignStatus.value) return
}

// 改为:
if (selectedSignStatus.value) {
  if (selectedSignStatus.value === '未签订') {
    // 未签订 = 所有非已签订合同（含红/橙/黄预警）
    if (contract.signStatus === '已签订') return
  } else {
    const statusVal = getWarningLabel(contract)
    if (statusVal !== selectedSignStatus.value) return
  }
}
```

## 验证方式
1. 从供应链看板点击"未签订"卡片下钻 → 报表显示所有未签订合同（含红/橙/黄预警），不仅限于超期<21天的

4. 选择"A"后表格只显示A级供应商的合同
5. 重置按钮清空供应商等级筛选
6. 若下钻传来 `supplierLevel: 'A'` 参数，自动选中A级筛选
