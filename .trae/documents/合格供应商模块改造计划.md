# 合格供应商模块改造计划

## 概述
将供应链看板中的"供应商统计"模块改造为"合格供应商"，增加A/B/C三级切换功能。

## 当前状态
- **数据源**: [mockData.js](src/data/mockData.js#L218-L227) 的 `supplierStatChart1Data`
  - title: `'供应商统计'`, subTitle: `'A级承包商统计'`
  - 只有A级数据（无B/C级）
- **组件**: [SupplierStatChart.vue](src/components/dashboard/SupplierStatChart.vue)（287行）
  - L5: 显示 `{{ title }}` 标题
  - L6: 显示 `{{ subTitle }}` 副标题
  - L26-32: 紫色信息栏显示"A级供应商数量X个，占比Y%"
  - L70-72: `isALevelChart` 通过 subTitle 判断是否为A级模式
  - 图表只展示A级数据

### 步骤3：改造 SupplierStatChart.vue 脚本部分

#### 3a. 新增响应式变量和计算属性
```js
// 等级切换相关
const selectedLevel = ref('A')           // 默认选中A
const levelOptions = computed(() => {
  const lvls = props.levels || {}
  return [
    { key: 'A', label: `A级供应商${lvls.A?.supplierCount || 0}家`, color: '#3b82f6' },
    { key: 'B', label: `B级供应商${lvls.B?.supplierCount || 0}家`, color: '#10b981' },
    { key: 'C', label: `C级供应商${lvls.C?.supplierCount || 0}家`, color: '#f59e0b' }
  ]
})

const totalSupplierCount = computed(() => {
  const lvls = props.levels || {}
  return (lvls.A?.supplierCount || 0) + (lvls.B?.supplierCount || 0) + (lvls.C?.supplierCount || 0)
})

const isQualifiedChart = computed(() => props.subTitle === '合格供应商统计')
```

#### 3b. 修改 isALevelChart / isTop10Chart 判断逻辑
```js
// 原 isALevelChart: subTitle === 'A级承包商统计'
// 改为:
const isALevelChart = computed(() => isQualifiedChart.value)

// isTop10Chart 保持不变
const isTop10Chart = computed(() => props.subTitle === '按合同额排名')
```

#### 3c. 添加 handleLevelChange 方法
```js
const handleLevelChange = (level) => {
  selectedLevel.value = level
  if (chart) initChart()
}
```

#### 3d. 修改 initChart 中 isALevelChart 分支的 series 构建逻辑
当前固定使用 `props.series`，改为根据 `selectedLevel` 从 `props.levels` 取对应颜色和数据：

```js
if (isALevelChart.value) {
  const levelData = props.levels?.[selectedLevel.value] || props.levels?.A || {}
  const levelColor = levelData.color || '#3b82f6'

  seriesData.push({
    name: `${selectedLevel.value}级供应商数量`,
    type: 'bar',
    data: levelData.supplierCount ? null : /* 根据分类的分布数据 */,
    itemStyle: { color: levelColor }
  })
  // ... 合同数、合同额同理，使用 levelData.contractCount / contractAmount
}
```

注意：当前数据结构中 `contractCount` 是数组（按分类），而 `supplierCount` 是单个总数。需要为每个等级补充按分类的供应商数量数组。

**简化方案**：直接用现有的 series 数据，根据 selectedLevel 改变柱子颜色和系列名称即可。或者更准确地在 mockData 中给每个等级提供完整的按分类分布数据。

#### 3e. 修改 tooltip.formatter
在 tooltip 中动态显示当前选中的等级名称：

```js
tooltip: {
  trigger: 'axis',
  formatter: (params) => {
    let result = `<div style="font-weight:bold;margin-bottom:6px;">${params[0].name}</div>`
    params.forEach(param => {
      result += `<div>${param.marker}${param.seriesName}: ${param.value}</div>`
    })
    // 追加等级信息
    if (isALevelChart.value && selectedLevel.value) {
      result += `<div style="margin-top:4px;font-size:11px;color:#888;">当前查看: ${selectedLevel.value}级供应商</div>`
    }
    return result
  }
}
```
