# 项目跟踪数据表筛选条件调整计划

## 概述
将项目跟踪数据表（ProjectTrackingReport.vue）的筛选条件从"公司 + 项目类型"改为仅"基层单位"一个筛选项，并使筛选逻辑实际生效（按 branch 字段过滤）。

## 现状分析
文件：[ProjectTrackingReport.vue](file:///e:/trae_demo_env/营收看板/src/components/market/ProjectTrackingReport.vue)

当前有两个筛选项：
1. **公司**（`filters.company`）：使用英文 value 代码（如 `pipe-network`、`ecology`），与表格数据的 `branch` 字段（中文值）不匹配，筛选逻辑实际上未生效。
2. **项目类型**（`filters.projectType`）：施工类/产品类/服务类，同样未实际过滤。

表格数据的 `branch` 字段包含的基层单位值：市政事业部、环境建设、区域事业部、城水管管道、水务管道、二次养护、运营养护、浦东供排水、生态环境事业部、管网运维事业部、设计咨询中心、水表厂、设备成套，末行为"合计"。

## 修改内容

### 模板部分
- 将"公司"筛选项替换为"基层单位"筛选项：标签改为"基层单位:"，`v-model` 改为 `filters.grassrootsUnit`，下拉选项改为与 `branch` 字段一致的中文名（value 直接用中文名），末行"合计"不作为可选项。
- 删除"项目类型"筛选项整块。
- 保留"重置"按钮。

### 脚本部分
- `filters` ref：移除 `company` 和 `projectType`，新增 `grassrootsUnit: ''`。
- `filteredData` computed：改为按 `filters.value.grassrootsUnit` 过滤 `item.branch`（"合计"行始终保留）。
- `resetFilters` 函数：重置为 `{ grassrootsUnit: '' }`。

## 验证步骤
1. 确认筛选区仅剩"基层单位"下拉 + 重置按钮。
2. 选择某个基层单位后，表格仅显示该单位行 + 合计行。
3. 点击重置后恢复全部数据。
