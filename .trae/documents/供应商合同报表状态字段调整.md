## Proposed Changes

### 改动 1：列顺序调整 — 合同状态移到预警状态前

**位置**：L191-269

将"合同状态"列（当前 L258-269）整体移动到"签订状态"列（L191-201）之前，使新顺序为：
1. 序号
2. **合同状态**（移至此处）
3. **预警状态**（原"签订状态"改名）
4. 项目编号
5. ...

### 改动 2：签订状态 → 预警状态（无预警显示为空）

**位置**：L191-201 列定义

列标题从 `label="签订状态"` 改为 `label="预警状态"`。

渲染逻辑改为：
```html
<template #default="scope">
  <span v-if="scope.row.signStatus !== '已签订' && scope.row.overdueDays >= 21"
    :class="getSignStatusClass(scope.row)">
    {{ getWarningLabel(scope.row) }}
  </span>
  <span v-else></span>
</template>
```

逻辑说明：
- 已签订 → 空（无预警）
- 未签订 + overdueDays ≥ 60 → 红色预警
- 未签订 + overdueDays ≥ 45 → 橙色预警
- 未签订 + overdueDays ≥ 21 → 黄色预警
- 未签订 + overdueDays < 21 → 空（无预警，普通未签订）

### 改动 5：默认排序实现

在 `tableData` computed 中（L1150 附近），对展开后的合同数据按预警级别排序。

新增排序权重函数：
```js
const getSortWeight = (row) => {
  if (row.signStatus === '已签订') {
    // 已签订：按合同状态排序
    const statusOrder = { '正常履约': 5, '履约结束': 6, '违约中': 7 }
    return statusOrder[row.contractStatus] || 8
  }
  // 未签订：按预警级别排序
  const days = row.overdueDays || 0
  if (days >= 60) return 1  // 红色预警
  if (days >= 45) return 2  // 橙色预警
  if (days >= 21) return 3  // 黄色预警
  return 4                   // 未签订（无预警）
}
```

在 tableData computed 的 return 前排序：
```js
result.sort((a, b) => getSortWeight(a) - getSortWeight(b))
```

## Assumptions & Decisions

1. "无预警则显示为空"包括两种情况：已签订合同、未签订但逾期天数<21天的合同
2. 合同状态的"未签订"是衍生值，不需要修改 rawData 中的 contractStatus 字段，而是在显示时动态判断
3. 排序在 tableData computed 中实现，影响最终表格数据的顺序
4. 签订状态筛选器选项不变（仍为5个选项），仅 label 文字改为"预警状态"
5. "违约中"状态保留在合同状态选项中，用户未要求删除

1. 打开供应商合同签订报表 → 确认列顺序为：序号、合同状态、预警状态、项目编号...
2. 已签订合同的"预警状态"列显示为空

6. 合同状态筛选器新增"未签订"选项，选择后能正确筛选
7. 默认表格数据排序为：红色预警 → 橙色预警 → 黄色预警 → 未签订 → 正常履约 → 履约结束
