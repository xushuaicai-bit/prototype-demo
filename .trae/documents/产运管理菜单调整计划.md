# 产运管理菜单调整计划

## 概述

在「产运管理」模块的侧边栏二级菜单中，保留现有 5 个菜单项，新增 5 个带三级子菜单的管理分类。

## 当前状态分析

### 现有「产运管理」菜单（Sidebar.vue 第 119-164 行）

当前包含 **5 个菜单项**：

| 序号 | 菜单名称    | 类型       | key                    |
| -- | ------- | -------- | ---------------------- |
| 1  | 营收管理看板  | 无子菜单     | revenue-dashboard      |
| 2  | 供应链管理看板 | 无子菜单     | supply-chain-dashboard |
| 3  | 生产看板    | 无子菜单     | production-dashboard   |
| 4  | 营收报表    | 有 3 个子菜单 | revenue-report         |
| 5  | 供应链报表   | 有 2 个子菜单 | supply-chain-report    |

### 涉及文件

* **`src/components/layout/Sidebar.vue`** — 唯一需要修改的文件，menuMap 中 `'产运管理'` 键对应的数组

## 修改方案

在 `Sidebar.vue` 的 `menuMap['产运管理']` 数组中，**保留现有 5 项不变**，在其后追加 5 个新的含 `children` 子菜单的菜单项：

### 新增菜单项明细

#### 1. 供应链管理（4 个三级菜单）

* 承包商/供应商库（key: `supplier-library`）

* 承包商/供应商选用（key: `supplier-selection`）

* 承包商/供应商合同（key: `supplier-contract-mgmt`）

* 集采对账单（key: `centralized-reconciliation`）

#### 2. 计量统计管理（2 个三级菜单）

* 产值营收管理（key: `output-revenue-mgmt`）

* 承包商/供应商计量（key: `supplier-measurement`）

#### 3. 生产管理（9 个三级菜单）

* 生产开项（key: `production-item`）

* 参建单位（key: `construction-unit`）

* 计量单位（key: `measurement-unit`）

* 进度管理（key: `progress-mgmt`）

* 风险管理（key: `risk-mgmt`）

* 险情管理（key: `danger-mgmt`）

* 质量管理（key: `quality-mgmt`）

* 设备管理（key: `equipment-mgmt`）

* 材料管理（key: `material-mgmt`）

#### 4. 技术管理（4 个三级菜单）

* 方案计划管理（key: `plan-mgmt`）

* 技术方案管理（key: `tech-solution-mgmt`）

* 资料库（key: `document-library`）

* 图章管理（key: `seal-mgmt`）

#### 5. 物资管理（5 个三级菜单）

* 材料计划管理（key: `material-plan-mgmt`）

* 材料领用供应（key: `material-requisition`）

* 材料库存管理（key: `material-inventory`）

* 材料对账管理（key: `material-reconciliation`）

* 材料统计报表（key: `material-statistics`）

### 实现方式

每个新菜单项遵循现有数据结构格式：

```javascript
{
  name: '菜单名称',
  icon: () => h('svg', { ... }, [ h('path', { ... }) ]),
  key: '唯一标识',
  children: [
    { name: '子菜单名', key: '子菜单标识' },
    // ...
  ]
}
```

图标复用现有模式（使用 Heroicons 风格的 SVG path），为每个新分类选择语义匹配的图标。

## 假设与决策

1. **保留现有菜单**：原有 5 个菜单项（营收管理看板、供应链管理看板、生产看板、营收报表、供应链报表）完全不动
2. **仅修改 Sidebar.vue**：不需要修改 Header.vue 或 App.vue，因为新菜单项暂无对应视图组件，点击后会走现有逻辑（viewMap 中无匹配则不渲染具体内容）
3. **图标风格统一**：使用与现有菜单一致的 SVG 图标风格

## 验证步骤

1. 启动开发服务器 `npm run dev`
2. 点击顶部导航「产运管理」
3. 确认侧边栏显示原有 5 项 + 新增 5 项共 10 个二级菜单
4. 依次展开每个新增菜单，确认三级子菜单数量和名称正确
5. 确认原有 5 个菜单功能不受影响\
   其他没提到的不要修改
