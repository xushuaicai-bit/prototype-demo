# 供应链看板图表调整计划

## 概述

对供应链看板中的3个组件进行UI优化：Top10供应商图增加双轴+图例、供应商统计增加图例+横轴旋转、合同签订及时性圆环放大。

## 当前状态分析

### 1. SupplierStatChart.vue（当前状态）

* **A级承包商模式**：已有双Y轴（数量左/金额右），但 `legend: { show: false }`，无图例

* **Top10模式**：所有 series 都是 bar 类型，单Y轴，`legend: { show: false }`

* 横轴：Top10 模式已 rotate: 30，但可能不够

* 图表高度：`h-64`（256px）

### 2. SupplyChainTimeliness.vue（当前状态）

* 4个分类子项，每个含 SVG 圆环进度条

* 圆环尺寸：`w-16 h-16`（64px），半径 r=40

* 布局：`grid-cols-4`，下方有大量空白

***

## 详细修改方案

### 调整1：Top10 供应商 — 双轴 + 单柱 + 折线 + 图例

**文件**: [SupplierStatChart.vue](src/components/dashboard/SupplierStatChart.vue)

#### 数据层变更（mockData.js）

当前 Top10 的 series 有2项：

```
[0] name: '履约中合同数' (bar)
[1] name: '履约中合同额' (bar) ← 需改为 line
```

无需改 mockData，只需在组件中根据名称判断渲染类型。

#### 组件变更

**Top10 模式的 initChart 分支**（第173-203行）改为：

```js
// Top10 模式
yAxis = [{ type: 'value', name: '数量' }]
yAxis.push({
  type: 'value',
  name: '金额(万元)',
  position: 'right',
  axisLabel: { formatter: '{value}' }
})

props.series.forEach((s, index) => {
  if (s.name.includes('合同额')) {
    // 折线系列
    seriesData.push({
      name: s.name,
      type: 'line',
      smooth: true,
      yAxisIndex: 1,
      data: s.data,
      itemStyle: { color: s.color },
      lineStyle: { color: s.color, width: 3 }
    })
  } else {
    // 柱状系列
    seriesData.push({
      name: s.name,
      type: 'bar',
      data: s.data,
      itemStyle: { color: s.color }
    })
  }
})
```

**开启 legend**：

* 第216-218行 `legend: { show: false }` → `legend: { show: true, bottom: 0, itemWidth: 14, itemHeight: 10, textStyle: { fontSize: 11 } }`

**横轴文字旋转加大**：

* 第229行 `rotate: isTop10Chart.value ? 30 : 0` → `rotate: isTop10Chart.value ? 35 : 0`

* grid.bottom 从 `'8%'` 调为 `'15%'` 给旋转标签留空间

***

### 调整2：供应商统计 — 开启图例

**文件**: [SupplierStatChart.vue](src/components/dashboard/SupplierStatChart.vue)

同一文件的 A级承包商模式，只需将全局 `legend: { show: false }` 改为 `show: true`。

legend 内容会自动从 series 的 name 字段生成：

* 深蓝色柱：`A级承包商数量`

* 浅蓝色柱：`履约中合同数（非集采）`

* 浅蓝色折线：`履约中合同额（非集采）`

右侧 Y轴名称加上单位：`name: '金额(万元)'`

**横轴旋转**：A级承包商模式的 categories 也加 rotate: 20（4个分类名较长时需要）

***

### 调整3：合同签订及时性 — 圆环放大铺满

**文件**: [SupplyChainTimeliness.vue](src/components/dashboard/SupplyChainTimelines.vue)

当前问题：

* SVG 圆环 `w-16 h-16`（64px）太小

* 下方有大量空白未利用

修改方案：

| 属性                  | 当前值                 | 新值                                    |
| ------------------- | ------------------- | ------------------------------------- |
| 圆环容器                | `w-16 h-16`         | `w-24 h-24`（96px）                     |
| SVG viewBox         | `100x100`, r=40     | 保持不变                                  |
| circle stroke-width | 8                   | 10（更粗更清晰）                             |
| 中心文字字号              | text-sm / text-xs   | text-base / text-sm                   |
| 整体布局                | `grid-cols-4 gap-4` | `grid-cols-4 gap-4` 不变，但让卡片 flex-1 铺满 |

同时给每个分类卡片添加 `flex-1` 让内容撑满可用高度。
