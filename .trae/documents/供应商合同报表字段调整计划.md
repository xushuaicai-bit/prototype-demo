# 供应商合同签订报表数据补全计划

## 问题诊断
用户反馈报表数据全部为空。经代码分析，根因如下：

### 致命问题
1. **`getWarningLabel()` 函数缺失** — 模板 L173 调用了 `getWarningLabel(scope.row)` 但 `<script>` 中从未定义该函数，导致运行时 ReferenceError，整个组件渲染失败/表格无数据
2. **rawData 中全部 ~45 个 contract 对象缺少 `contractStartDate` 和 `contractEndDate` 字段** — 新增的两列（合同开始时间、合同计划/实际结束时间）无数据可绑定

### 次要问题（遗留清理项）
3. 筛选栏标签仍为"签订状态："（L95），应改为"预警等级："
4. `allSignStatus` 选项仍为 `['已签订', '未签订']`（L327），缺少红色/橙色/黄色预警选项
5. 存在大量无用死代码：`selectedSupplier`、`selectedContractStatus`、`allSuppliers`、`allContractStatus`、`getLevelClass()`、`getContractStatusClass()` 及相关筛选逻辑

---

## 修改步骤

### 步骤1：添加 getWarningLabel 函数（致命修复）
**位置**: [SupplierContractReport.vue](src/components/supply-chain/SupplierContractReport.vue#L1017) — 在 `getSignStatusClass` 函数之后插入
```js
const getWarningLabel = (row) => {
  if (row.signStatus === '已签订') return '已签订'
  const days = row.overdueDays
  if (days >= 60) return '红色预警'
  if (days >= 45) return '橙色预警'
  if (days >= 21) return '黄色预警'
  return '未签订'
}
```

### 步骤2：为所有 contract 对象补充 contractStartDate / contractEndDate（致命修复）
**位置**: [SupplierContractReport.vue](src/components/supply-chain/SupplierContractReport.vue#L341-L899)

为每个 contract 对象添加两个字段。数据生成规则：
- **已签订合同** (`signStatus === '已签订'`)：
  - `contractStartDate` = `actualDate` 或 `plannedDate` + 若干天（合同开始≈实际签订或稍后）
  - `contractEndDate` = 合理的未来日期（根据金额/工程量推算工期）
- **未签订合同** (`signStatus === '未签订'`)：
  - `contractStartDate` = 空字符串 `''`
  - `contractEndDate` = 空字符串 `''`

涉及10个项目共约45个 contract 对象，逐个补充。

### 步骤3：筛选栏标签同步修改
- L95: `签订状态：` → `预警等级：`
- L98: placeholder `请选择签订状态` → `请选择预警等级`
