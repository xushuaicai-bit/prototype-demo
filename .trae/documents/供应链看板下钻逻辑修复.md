### 发现的问题

***

### BUG-1 \[严重] — 合同图表柱状图单位点击后 unit 被覆盖

**位置**: [UnifiedDashboard.vue#L1039-L1044](src/views/UnifiedDashboard.vue#L1039-L1044)

**现象**: `SupplyContractChart.vue` 在**单位模式(柱状图)** 下点击某个单位柱子时，emit 参数为 `{ unit: '管网事业部' }`。但在 `handleScDrillDown` 中：

```js
const handleScDrillDown = (filter) => {
  emit('navigate', { 
    view: 'supply-chain', 
    report: 'supplier-contract', 
    filter: { ...filter, unit: selectedFilter.value }  // ← unit 被全局筛选值覆盖!
  })
}
```

展开运算符 `...filter` 先放入了 `unit: '管网事业部'`，紧接着 `unit: selectedFilter.value` 把它覆盖成了 `'全部'`（或当前选中的全局筛选）。**用户点击的具体单位信息丢失。**

**影响的下钻场景**: 合同签订情况图表 → 单位模式 → 点击任意单位的柱子 → 应跳转到该单位的合同报表，实际跳转后单位筛选为"全部"

**修复方案**: 当 filter 中已携带 unit 时，优先保留 filter 中的值而非用全局 selectedFilter 覆盖：

```js
const handleScDrillDown = (filter) => {
  const finalUnit = filter.unit || selectedFilter.value
  emit('navigate', { 
    view: 'supply-chain', 
    report: 'supplier-contract', 
    filter: { ...filter, unit: finalUnit } 
  })
}
```

### BUG-2 \[中等] — SupplierContractReport 缺少 supplierName 字段映射

**位置**: [SupplierContractReport.vue#L1093-L1130](src/components/supply-chain/SupplierContractReport.vue#L1093-L1130)

**现象**: `applyInitialFilter()` 函数支持 category/signStatus/overdueDays/unit/contractStatus 五个字段的映射，但**缺少 supplierName**。虽然当前主链路中 Top10 供应商下钻正确路由到了 `supplier-detail`（SupplierDetailReport），但如果未来有任何入口需要按供应商名称筛选合同报表，该字段将静默失效。且 SupplierContractReport 自身已有 `selectedSupplier` 状态和供应商下拉框 UI（L96-109），说明组件层面是支持按供应商筛选的。

**修复方案**: 在 applyInitialFilter 中补充 supplierName 映射：

```js
if (props.initialFilter.supplierName) {
  selectedSupplier.value = props.initialFilter.supplierName
}
```

同步在 sessionStorage 回退分支中也补充该字段（L1111-1128）。

## Assumptions & Decisions

1. 假设 `selectedSupplier` 是 SupplierContractReport 中供应商下拉框绑定的状态变量（需通过代码确认变量名准确）
2. 不统一 SupplyChainReport 中两个子组件的 prop 名称（initialFilter vs reportFilter），因为它们各自内部定义不同，强行统一反而会增加改动面
3. App.vue 的 `handleDrillDown` 空实现暂不处理——当前所有供应链下钻都走 `navigate` 事件，不经过此路径
