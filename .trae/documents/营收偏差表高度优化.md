# 营收偏差情况表高度优化计划

## 一、当前状态分析

### 现有布局（RevenueDashboard.vue L301 / UnifiedDashboard.vue L305）

```
max-h-[320px] overflow-y-auto  ← 滚动容器，硬编码320px
├── 汇总行（偏差项目总数/涉及单位）~32px
└── 9条单位列表（space-y-2间距）
    ├── 每条：py-2 px-3 → 约30px高
    └── 间距：8px
    总计：~366px > 320px → 需滚动
```

**问题**：9家单位在320px内放不下，最后1~2条需要滚动才能看到。

### 数据量
- 共 **9条** 基层单位数据，每条包含：名称、颜色圆点、重点项目数、一般项目数、总偏差额

---

## 二、方案对比

| 方案 | 描述 | 高度预估 | 优缺点 |
|------|------|----------|--------|
| **A. 紧凑单列** | 缩减padding/字号/间距，一行内塞入更多信息 | ~220px可放全9条 | 信息密度高，但字号偏小 |
| **B. 双列网格** | 9条拆为左5右4两列显示 | 高度降约50% | 视觉整齐，空间利用率最高 |
| **C. 默认展示4条+展开** | 默认前4条，底部"查看剩余5家"按钮切换 | ~180px（默认态） | 交互清晰，但需额外点击 |

### 推荐：方案B — 双列网格布局

理由：
1. **高度直接砍半**：从单列~366px变为双列~190px，完美匹配"用当前1/2高度"的需求
2. **无需隐藏信息**：9家单位全部可见，不需要"展开/收起"交互
3. **视觉更整齐**：与右侧两列图表等高对齐效果更好
4. **改动最小**：只需改模板CSS类，不动数据逻辑

## 三、具体修改方案（方案B：双列紧凑网格）

### 修改文件
1. `e:\trae_demo_env\营收看板\src\views\RevenueDashboard.vue` （L299-L340）
2. `e:\trae_demo_env\营收看板\src\views\UnifiedDashboard.vue` （L303-L344）

### 改动内容

#### 3.1 滚动容器：去掉 max-h 限制，改为自适应

```html
<!-- 改前 -->
<div class="max-h-[320px] overflow-y-auto pr-1">

<!-- 改后 -->
<div class="grid grid-cols-2 gap-x-4 gap-y-1.5">
```

- 去掉 `max-h-[320px] overflow-y-auto`（不再需要滚动）
- 改为 `grid grid-cols-2` 双列网格
- `gap-x-4` 列间距16px，`gap-y-1.5` 行间距6px（比原来8px更紧凑）

#### 3.2 汇总行：保留但调整样式

保持汇总行不变（仍在网格上方），仅微调 margin。

#### 3.3 单位条目：紧凑化

```html
<!-- 改前 -->
<div class="group flex items-center justify-between py-2 px-3 rounded-lg hover:bg-gray-50 ...">
  <!-- 左侧：圆点 + 名称（占满剩余宽度） -->
  <div class="flex items-center flex-1 min-w-0">...</div>
  <!-- 右侧：标签 + 金额（横向排列） -->
  <div class="flex items-center gap-3 ml-3 flex-shrink-0">...</div>
</div>

<!-- 改后 -->
<div class="group flex items-center justify-between py-1.5 px-2.5 rounded-md hover:bg-gray-50 ...">
  <!-- 左侧：圆点 + 名称 -->
  <div class="flex items-center flex-1 min-w-0">
    <span class="w-2 h-2 rounded-full mr-1.5 flex-shrink-0" ...></span>   <!-- 圆点缩小 10→8px -->
    <span class="text-xs text-gray-700 truncate">{{ unit.name }}</span>     <!-- 字号 14→12px，截断不换行 -->
  </div>
  <!-- 右侧：标签 + 金额 -->
  <div class="flex items-center gap-1.5 ml-2 flex-shrink-0">               <!-- 间距缩小 -->
    <span class="text-[11px] px-1 py-px rounded ...">重点{{ unit.keyProjectCount }}</span>   <!-- 更小标签 -->
    <span class="text-[11px] px-1 py-px rounded ...">一般{{ unit.generalProjectCount }}</span>
    <span class="text-xs font-bold ...">{{ formatAmount(unit.totalAmount) }}万</span>
  </div>
</div>
```

关键变化：
| 属性 | 改前 | 改后 | 原因 |
|------|------|------|------|
| 容器 | `max-h-[320px] overflow-y-auto` 单列 | `grid grid-cols-2 gap-x-4 gap-y-1.5` | 双列布局 |
| 条目padding | `py-2 px-3` | `py-1.5 px-2.5` | 减少垂直空间 |
| 圆点尺寸 | `w-2.5 h-2.5` | `w-2 h-2` | 配合整体缩小 |
| 名称字号 | `text-sm`(14px) | `text-xs`(12px) | 节省空间 |
| 名称换行 | `whitespace-normal` | `truncate` | 双列中名称截断更整洁 |
| 标签字号 | `text-xs`(12px) | `text-[11px]` | 更紧凑 |
| 标签padding | `px-1.5 py-0.5` | `px-1 py-px` | 最小化标签 |
| 右侧间距 | `gap-3` | `gap-1.5` | 元素靠拢 |
| 圆角 | `rounded-lg`(8px) | `rounded-md`(6px) | 略微调小 |

#### 3.4 预估高度

双列布局下：
- 左列5条 + 右列4条 = 最高5行
- 每条约22px（py-1.5=12px + 内容~10px）+ gap-y-1.5=6px
- 总计：5 × 22 + 4 × 6 = **134px**
- 加上标题栏(~44px) + 汇总行(~28px) = **~206px**

相比原来的 ~366px，**高度降低约44%**，完全可以在第二排三列等高布局中良好适配。

## 四、备选方案（若用户选择其他）

### 方案A（紧凑单列）改动要点：
- 保持单列 `overflow-y-auto`
- `max-h-[320px]` → `max-h-[220px]`
- 所有padding/字号/间距同上表缩减
- 9条数据在220px内勉强放下或轻微滚动

### 方案C（默认4条+展开）改动要点：
- 用 `slice(0, 4)` 控制默认展示数量
- 新增 `expanded` ref 状态
- 底部加"查看剩余 N 家 ▾"按钮
- 展开后恢复完整列表 + 滚动
