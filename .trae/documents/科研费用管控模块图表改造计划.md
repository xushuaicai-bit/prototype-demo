# 科研费用管控模块图表改造计划

## 一、需求摘要

将 `TechResearchDashboard.vue` 中的"科研费用管控"模块从文字+进度条展示，改造为 ECharts 柱状图：
- 两根柱子：[专项]、[配套]，横轴为基层单位
- 柱高 = 实际完成值；柱顶水平线 = 计划目标值（用 custom series 绘制，不使用普通柱状图展示计划值）
- 单位：万元
- 数据标签：实际值、完成率、偏差
- 颜色：达标(>=100%)突出，未达标(<100%)弱化+风险提示
- 计划线使用高亮辅助色
- Hover 显示浮层（基层单位/计划值/实际值/完成率/偏差）
- 点击单位：alert 提示（暂不跳转）
- 视觉优化：简化坐标轴、减少网格线、强调计划与实际差距、保留留白

### 5.2 新增函数（在 `handleFilterChange` 之前插入）

#### 颜色常量
```js
const EXP_COLOR = {
  special: '#3b82f6',
  support: '#10b981',
  mutedSpecial: '#bfdbfe',
  mutedSupport: '#a7f3d0',
  planLine: '#f59e0b',
  risk: '#ef4444'
}
const EXP_BAR_WIDTH = 20
```

#### buildExpenseData(quarter)
返回 `{ categories, specialPlan, specialActual, supportPlan, supportActual, rates, deviations }`：
- 从 `researchData.expenses.byCompany[quarter]` 读取
- categories = 各单位 name
- specialPlan/Actual = item.special.plan/actual
- supportPlan/Actual = item.supporting.plan/actual
- rate = actual/plan*100（plan=0 时若 actual>0 算 100，否则 0）
- deviation = actual - plan

#### initExpenseChart()
```js
const initExpenseChart = () => {
  if (!expenseChartRef.value) return
  expenseChart = echarts.init(expenseChartRef.value)
  updateExpenseChart()
  expenseChart.on('click', (params) => {
    if (params.componentSubType === 'bar') {
      const d = buildExpenseData(selectedQuarter.value)
      const idx = d.categories.indexOf(params.name)
      if (idx >= 0) {
        const type = params.seriesName === '专项' ? 'special' : 'supporting'
        const item = researchData.expenses.byCompany[selectedQuarter.value][idx]
        window.alert(
          '基层单位：' + params.name + '\n' +
          '类型：' + params.seriesName + '\n' +
          '计划值：' + item[type].plan + '万\n' +
          '实际值：' + item[type].actual + '万\n' +
          '完成率：' + (item[type].plan ? Math.round(item[type].actual / item[type].plan * 100) : 0) + '%\n' +
          '偏差：' + (item[type].actual - item[type].plan) + '万'
        )
      }
    }
  })
}
```

#### updateExpenseChart()
核心实现，包含：
1. 调用 buildExpenseData 获取数据
2. 两个 bar 系列：专项(实际)、配套(实际)，barWidth=20，barGap='30%'
3. 一个 custom 系列：renderItem 为每个柱子绘制水平计划线
4. label formatter 使用 rich text 显示：实际值、完成率、偏差
5. 颜色逻辑：rate>=100 使用主色，<100 使用 muted 色
6. tooltip 显示完整信息
7. 视觉优化：yAxis 隐藏分隔线、axisLine；xAxis 简化；grid 增加留白

**custom series renderItem 关键逻辑**：
```js
renderItem: (params, api) => {
  const idx = api.value(0)  // 单位索引
  const type = api.value(1) // 0=专项, 1=配套
  const planVal = api.value(2)
  const point = api.coord([idx, planVal])
  const side = type === 0 ? -1 : 1
  const offset = 13 * side
  const half = 10
  return {
    type: 'group',
    children: [
      {
        type: 'line',
        shape: {
          x1: point[0] + offset - half, y1: point[1],
          x2: point[0] + offset + half, y2: point[1]
        },
        style: { stroke: EXP_COLOR.planLine, lineWidth: 2 }
      }
    ]
  }
}
```

**数据准备**：
- custom series data = 遍历每个单位 × [专项,配套]，每个 item = [unitIndex, typeIndex, planValue]
- 配套计划线只画配套的，专项计划线只画专项的

**label 配置**（bar series）：
```js
label: {
  show: true,
  position: 'top',
  distance: 8,
  formatter: (params) => {
    const d = buildExpenseData(selectedQuarter.value)
    const idx = params.dataIndex
    const type = params.seriesName === '专项' ? 'special' : 'supporting'
    const item = researchData.expenses.byCompany[selectedQuarter.value][idx]
    const plan = item[type].plan
    const actual = item[type].actual
    const rate = plan ? Math.round(actual / plan * 100) : 0
    const dev = actual - plan
    const NL = String.fromCharCode(10)
    const rateColor = rate >= 100 ? '#16a34a' : '#ef4444'
    const devColor = dev >= 0 ? '#16a34a' : '#ef4444'
    return '{actual|' + actual + '万}' + NL +
           '{rate' + (rate>=100?'Ok':'Risk') + '|完成率 ' + rate + '%}' + NL +
           '{dev' + (dev>=0?'Ok':'Risk') + '|偏差 ' + (dev>0?'+':'') + dev + '万}'
  },
  rich: {
    actual: { color: '#1f2937', fontSize: 11, fontWeight: 'bold' },
    rateOk: { color: '#16a34a', fontSize: 10 },
    rateRisk: { color: '#ef4444', fontSize: 10 },
    devOk: { color: '#16a34a', fontSize: 10 },
    devRisk: { color: '#ef4444', fontSize: 10 }
  }
}
```

**颜色逻辑**（itemStyle 为每条数据动态着色）：
```js
data: data.map((item, idx) => {
  const it = researchData.expenses.byCompany[selectedQuarter.value][idx]
  const plan = it.special.plan
  const actual = it.special.actual
  const rate = plan ? actual / plan : 0
  return {
    value: actual,

  }

```

**tooltip**：
```js
tooltip: {
  trigger: 'axis',
  axisPointer: { type: 'shadow' },
  formatter: (params) => {
    const idx = params[0].dataIndex
    const it = researchData.expenses.byCompany[selectedQuarter.value][idx]
    let html = '<div style="font-weight:bold;margin-bottom:4px">' + it.name + '</div>'
    ;[['专项','special'],['配套','supporting']].forEach(([label, key]) => {
      const p = it[key].plan
      const a = it[key].actual
      const r = p ? Math.round(a/p*100) : 0
      const d = a - p
      html += label + '：计划 ' + p + '万 / 实际 ' + a + '万 / 完成率 ' + r + '% / 偏差 ' + (d>0?'+':'') + d + '万<br/>'
    })
    return html
  }
}
```

## 七、假设与决策

1. **布局**：科研费用管控卡片整行满宽展示（用户明确要求）
2. **点击交互**：点击柱体仅 `window.alert` 提示，不做页面跳转（用户明确要求"暂不跳转仅提示"）
3. **计划线绘制**：使用 ECharts custom series 的 renderItem 绘制每个柱子上方的水平短线段，而非使用 markLine（markLine 无法按单位+类型分别绘制）
4. **柱宽**：barWidth=20，柱间距 barGap='30%'，custom 线段 half=10、offset=13*side
5. **Q4 actual=0**：会显示为高度 0 的柱体，完成率 0%，偏差负值，颜色弱化+风险色标签，符合"未达标"展示
6. **保留 currentExpense computed**：虽然不再用于模板展示，但不删除避免引入未定义变量错误（若模板无引用可保留）
