# 项目跟踪数据表二级表头改造计划

## 概述
将项目跟踪数据表的 AAA、AA、A、小计列改为二级表头结构（项目数量 + 项目金额），移除占比列，确保底部合计栏存在。

## 当前状态分析

文件：`src/components/market/ProjectTrackingReport.vue`

当前表结构为单行表头：
- 序号 | 基层单位 | AAA | AA | A | 小计 | 占比
- AAA/AA/A/小计列在单个单元格内用 `flex flex-col` 上下堆叠显示数量和金额
- 数据结构为嵌套对象：`aaa: { count: 5, amount: 50000000 }`
- 已有"合计"行作为数据最后一行
- 有占比列和 `percentage` 字段

## 修改方案

### 1. 模板修改 — 二级表头

将 AAA、AA、A、小计列改为嵌套 `el-table-column`，每个主列下包含两个子列：

```
| 序号 | 基层单位 | AAA                  | AA                   | A                    | 小计                 |
|      |         | 项目数量 | 项目金额   | 项目数量 | 项目金额    | 项目数量 | 项目金额    | 项目数量 | 项目金额    |
```

每个子列使用 `template #default` 读取 `scope.row.xxx.count` 和 `scope.row.xxx.amount`，与现有数据结构一致。

示例（AAA列）：
```html
<el-table-column label="AAA" :header-cell-style="{ backgroundColor: '#4A8AC4' }">
  <el-table-column label="项目数量" width="100" align="center">
    <template #default="scope">
      <span class="font-medium">{{ scope.row.aaa.count }}</span>
    </template>
  </el-table-column>
  <el-table-column label="项目金额" width="120" align="center">
    <template #default="scope">
      <span class="text-xs text-gray-500">{{ formatNumber(scope.row.aaa.amount) }}</span>
    </template>
  </el-table-column>
</el-table-column>
```

对 AA、A、小计列同理处理。小计列的金额和数量用蓝色加粗样式突出。

### 2. 移除占比列

删除模板中占比的 `el-table-column`（L136-144）。

### 3. 数据修改

- 从 `tableData` 每行中移除 `percentage` 字段
- 保留"合计"行（已有，无需新增）

### 4. 合计行样式

使用 `:row-class-name` 为"合计"行添加高亮样式（加粗、浅蓝背景），与其他数据行区分。

### 5. 记录数修正

底部"共 X 条记录"改为 `filteredData.length - 1`（不含合计行），已有逻辑无需修改。

## 不修改的部分
- 筛选区域（公司、年份、项目类型）— 保持不变
- `formatNumber` 函数 — 保持不变
- `filteredData` 逻辑 — 保持不变
- 导出按钮 — 保持不变

## 假设与决策
1. **数据结构不变**：保持 `aaa: { count, amount }` 嵌套结构，用 template 读取。
2. **合计行**：保留现有数据中的合计行，通过 `row-class-name` 添加样式。
3. **子列宽度**：项目数量列 100px，项目金额列 120px，保持表格紧凑。
4. **单位标注**：标题旁已有"单位：元"标注，表头不再重复。

## 验证步骤
1. 打开 http://localhost:5173/ → 市场报表 → 项目跟踪数据表
2. 确认表头为两行：首行 AAA/AA/A/小计，次行 项目数量/项目金额
3. 确认数据正确显示在对应子列中
4. 确认占比列已完全移除
5. 确认底部合计行存在且样式突出（加粗/背景色）
6. 确认筛选功能正常工作
