# 当年累计营收下钻功能修复计划

## Summary

修复营收管理看板中"当年累计营收"列表的下钻功能：点击"产品类"应跳转到营收汇总表的"产品类总营收"Tab，点击"其他类"应跳转到"其他类总营收"Tab（原"其他业态总营收"）。

**根因**：`handleSummaryDrill` 函数中的 `tabMap` 映射值与 `RevenueSummary.vue` 的 `el-tabs` 的 `name` 属性不匹配。

## Current State Analysis

### 发送端（RevenueDashboard.vue / UnifiedDashboard.vue）

**文件**: `RevenueDashboard.vue` L441-L455, `UnifiedDashboard.vue` L633-L647

```javascript
const handleSummaryDrill = (tabType) => {
  const tabMap = {
    '施工类': 'construction',        // ✅ 匹配
    '产品类': 'product-revenue',     // ❌ 应为 'product'
    '其他业态类': 'other-revenue'    // ❌ 应为 'other'，且数据中name是'其他类'
  }
  emit('navigate', { view: 'revenue', report: 'revenue-summary', filter: { ... } })
}
```

### 问题链路

| 用户点击 | item.name | tabMap 查找 | 发送的 tab 值 | RevenueSummary 匹配 | 结果 |
|---------|-----------|------------|-------------|-------------------|------|
| 施工类 | `'施工类'` | ✅ 找到 | `'construction'` | ✅ name="construction" | **正常** |
| 产品类 | `'产品类'` | ✅ 找到 | `'product-revenue'` | ❌ 无此 tab | **回退默认 construction** |
| 其他类 | `'其他类'` | ❌ **未找到**（key是`其他业态类`） | `undefined → 'construction'` | construction | **错误** |

## Verification Steps

1. 打开营收管理看板 / 产运看板
2. 在"总体营收情况"→"当年累计营收"列表中：
   - 点击"**产品类**" → 应跳转到营收报表 → 自动选中"**产品类总营收**"Tab
   - 点击"**其他类**" → 应跳转到营收报表 → 自动选中"**其他类总营收**"Tab
   - 点击"**施工类**" → 应跳转到营收报表 → 自动选中"**施工类总营收**"Tab（保持不变）
3. 确认无语法错误（GetDiagnostics）
