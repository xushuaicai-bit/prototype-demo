# 市场看板调整计划

## 概述
对市场看板及相关报表进行5项调整：版块位置调换、列名修改、新增字段、表格合并。

## 当前状态分析

### 市场看板 (MarketDashboard.vue) 布局
- **第一行 (3列)**：客户管理 | 重点跟踪项目 | 市场跟踪（数量+金额）
- **第二行 (3列)**：项目立项（数量+金额） | 项目投标（数量+金额） | 总包合同（数量+金额）

### 涉及文件
| 文件 | 用途 |
|------|------|
| `src/views/MarketDashboard.vue` | 市场管理看板 |
| `src/components/market/KeyProjectListReport.vue` | 重点跟踪项目清单报表 |
| `src/components/market/BidSummaryReport.vue` | 城市环境投标情况汇总报表 |
| `src/components/market/ProjectTrackingReport.vue` | 项目跟踪数据表组件（在MarketReport中使用） |
| `src/views/MarketTracking.vue` | 市场跟踪管理独立页面 |
| `src/views/MarketReport.vue` | 市场报表容器（根据子菜单切换组件） |
| `src/App.vue` | 主应用，包含路由逻辑 |
| `src/components/layout/Sidebar.vue` | 侧边栏菜单 |

## 修改方案

### 1. 市场看板版块位置调换 (MarketDashboard.vue)

**第一行调换**：重点跟踪项目 ↔ 市场跟踪（数量+金额）
- 将"重点跟踪项目"区块（L74-77）与"市场跟踪（数量+金额）"区块（L79-113）整体交换位置
- 调换后：客户管理 | 市场跟踪（数量+金额） | 重点跟踪项目

**第二行调换**：项目立项 ↔ 项目投标
- 将"项目立项（数量+金额）"区块（L117-165）与"项目投标（数量+金额）"区块（L167-201）整体交换位置
- 调换后：项目投标（数量+金额） | 项目立项（数量+金额） | 总包合同（数量+金额）

**注意**：仅交换模板中 div 区块的位置，不改变 script 中的图表逻辑。Vue ref 绑定会自动跟随 DOM 元素，图表初始化不受影响。

### 2. 重点跟踪项目清单修改 (KeyProjectListReport.vue)

**2a. 列名修改**：将"账户单位"改为"核算组织"
- L85: `label="账户单位"` → `label="核算组织"`
- prop 保持 `accountUnit` 不变

**2b. 新增列**：在"项目名称"后添加"基层单位"列
- 在 L61（项目名称列）之后插入新的 el-table-column：
  - `label="基层单位"`, `prop="grassrootsUnit"`, `width="120"`, `align="center"`
- 在 tableData 每条数据中添加 `grassrootsUnit` 字段

### 3. 城市环境投标情况汇总修改 (BidSummaryReport.vue)

**新增两列**：在"序号"后添加"基层单位"和"年份"
- **表头**：在序号 th（L26）之后插入两个 `rowspan="2"` 的 th：
  - `<th rowspan="2" class="...">基层单位</th>`
  - `<th rowspan="2" class="...">年份</th>`
- **表体**：在序号 td（L49）之后插入两个 td：
  - `{{ row.grassrootsUnit }}`
  - `{{ row.year }}`
- 在 tableData 每条数据中添加 `grassrootsUnit` 和 `year` 字段

### 4. 项目跟踪数据表列名修改 (ProjectTrackingReport.vue)

**列名修改**：将"分公司"改为"基层单位"
- L36: `label="分公司"` → `label="基层单位"`
- prop 保持 `branch` 不变

### 5. 市场跟踪管理与项目跟踪数据表合并

**目标**：将两个表合并为一个"项目跟踪数据表"，保留市场跟踪管理的筛选功能和占比列。

**5a. 增强 ProjectTrackingReport.vue**（作为合并后的组件）：
- 添加筛选区域（公司、年份、项目类型）— 从 MarketTracking.vue 移植
- 添加"占比"列 — 从 MarketTracking.vue 移植
- 列名"分公司"改为"基层单位"（与第4点一致）
- 添加 `filteredData` computed 和 `resetFilters` 方法
- 合并数据：使用 MarketTracking.vue 的数据（含 percentage 字段），同时修正"城乡管道"为"城水管管道"以保持一致性
- 页面标题保持"项目跟踪数据表"

**5b. 移除 MarketTracking.vue 独立页面**：
- 删除 `src/views/MarketTracking.vue` 文件

**5c. 更新 App.vue**：
- L35: 移除 `&& currentSubMenu !== 'market-tracking'` 条件，改为 `v-else-if="currentMenu === '市场报表'"`
- L41: 移除 MarketTracking 的条件渲染行
- L74: 移除 `import MarketTracking from './views/MarketTracking.vue'`

**5d. 更新 Sidebar.vue**：
- L114: 移除 `{ name: '市场跟踪管理', key: 'market-tracking' }` 菜单项

## 假设与决策
1. **图表逻辑不变**：看板版块调换仅改变模板 DOM 顺序，不修改 echarts 初始化逻辑，ref 绑定自动跟随。
2. **合并基准**：以 ProjectTrackingReport.vue 组件为合并目标（因为它在 MarketReport 容器中使用，符合现有架构），将 MarketTracking 的筛选和占比功能移植过来。
3. **数据一致性**：合并后使用 MarketTracking 的数据集（含占比），统一"城水管管道"名称。
4. **Mock 数据**：新增字段（基层单位、年份）使用合理的 Mock 数据填充。
5. **prop 名不变**：仅修改 label 显示文本，不改变数据字段名（如 accountUnit、branch），减少改动范围。
