# 材料集采合同情况图表改造

## 摘要
将材料集采合同情况图表从"柱+折线"模式改为"大标签统计+柱+多折线"模式。横轴从集采项目改为主材名称，点击"股份/环境"切换主材列表。柱子显示合同签订数量，3条折线显示合同签订金额、实际采购金额、已支付金额。

## 当前状态分析

### MaterialProcurementChart.vue（228行）
- **模板**：顶部有"集采/集租"Tab + 分类按钮 + 统计信息条 + 图表容器
- **ECharts**：xAxis=集采项目（股份/环境），左Y轴=数量（柱），右Y轴=完成率%（折线）
- **series**：3条（采购合同签订数量-柱、项目合同执行数量-柱、集采完成率-折线）
- **分类按钮**：读取 `props.categoryOptions[activeTab]`，但 mockData 未定义该字段，按钮实际为空

### mockData.js（L271-291）
```js
materialProcurementData = {

}
```

### UnifiedDashboard.vue（L444-450, L1082）
- `procurementData = materialProcurementData`（直接引用）
- 传入 props: title, categories, series, categoryOptions, currentUnit

### 3. MaterialProcurementChart.vue — 脚本改造

**新增 props**：`summary`（Object）、`groups`（Object）
**新增 ref**：`activeGroup`（默认 '股份'）
**新增 computed**：
- `groupKeys`：`Object.keys(props.groups || {})` → `['股份', '环境']`
- `currentGroup`：`props.groups?.[activeGroup.value]`
- `currentMaterials`：`currentGroup.value?.materials || []`
- `currentSummary`：`props.summary?.[activeGroup.value + '集采项目'] || {}`

**新增方法**：
- `switchGroup(key)`：设置 activeGroup，调用 initChart()

**ECharts 配置改造**（initChart）：
- xAxis.data = currentMaterials（主材名称）
- yAxis：左轴=数量，右轴=金额(万元)
- series：
  - 柱子：合同签订数量（currentGroup.contractCount），颜色 #3b82f6
  - 折线1：合同签订金额（currentGroup.contractAmount），颜色 #f59e0b
  - 折线2：实际采购金额（currentGroup.actualAmount），颜色 #10b981
  - 折线3：已支付金额（currentGroup.paidAmount），颜色 #ef4444

**删除**：原 displaySeries、getDisplayCategories、procurementStats、activeCategory、currentCategoryOptions 相关逻辑

## 假设与决策
- 保留"集采/集租"Tab，但图表内容仅响应"股份/环境"分组切换
- 大标签显示当前分组的汇总值（签订数量、执行数量、完成率）
- 柱子=合同签订数量（左Y轴），3条折线=合同签订金额/实际采购金额/已支付金额（右Y轴，单位万元）
- 删除原 `window.open` 跳转逻辑（用户未提及保留）
- 统计信息条改为大标签卡片形式
- 数据值为模拟数据，数量级合理即可
