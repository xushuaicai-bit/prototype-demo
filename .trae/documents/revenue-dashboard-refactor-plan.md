对 `RevenueDashboard.vue` 营收管理看板进行以下改造：

1. 名称修改

## 四、假设与决策

1. **不引入新依赖**: 所有UI组件均使用已有的 Tailwind CSS + Element Plus 风格，不安装新的npm包
2. **Mock数据内联**: 新增的数据直接写在组件内部或从 mockData.js 导入，不新建独立数据文件
3. **SupplyChainTimeliness 样式复用**: 改动4采用相似的结构风格，但数据独立，不直接引用该组件（因为数据结构不同）
4. **区域下钻功能**: 改动6.3的区域下钻（点击展开子区域）在tooltip中实现提示信息展示，暂不做完整交互
5. **年月选择器**: 使用原生 select 下拉或简单的按钮组实现，避免引入重量级日期选择器
