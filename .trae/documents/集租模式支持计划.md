# 集租模式支持计划

## 摘要
切换到"集租"Tab时，横轴显示"塔吊、施工电梯"，且只有股份集租（无环境集租按钮）。

## 当前状态
- **mockData.js** `materialProcurementData`：只有 `groups`（集采数据：股份7种主材 + 环境3种主材），无集租数据
- **MaterialProcurementChart.vue**：
  - `activeTab`（集采/集租）切换时调用 `initChart()`，但图表数据来源 `currentGroup` 读取的是同一个 `props.groups`
  - `groupKeys` = `Object.keys(props.groups)` = `['股份', '环境']`，不区分集采/集租
  - 大标签 `currentSummary` 用 `activeGroup + '集采项目'` 拼 key，集租时也读集采数据

## 改动方案

### 1. mockData.js — 新增集租数据（L271-295）

在 `materialProcurementData` 中新增 `rentGroups` 和 `rentSummary`：

```js
export const materialProcurementData = {
  title: '材料集采合同情况',
  summary: { ... },  // 集采汇总（不变）
  groups: { ... },   // 集采分组（不变）
  // 集租汇总
  rentSummary: {
    '股份集租项目': { signedCount: 85, executingCount: 72, completionRate: 85 }
  },
  // 集租分组（只有股份，无环境）
  rentGroups: {
    '股份': {
      materials: ['塔吊', '施工电梯'],
      contractCount: [12, 8],
      contractAmount: [5800, 3200],
      actualAmount: [5200, 2800],
      paidAmount: [4500, 2200]
    }
  }
}
```

### 2. MaterialProcurementChart.vue — 脚本改造

**新增 ref**：`activeRentGroup`（默认 '股份'）

**修改 computed**：
- `currentGroups`：`activeTab === '集采' ? props.groups : props.rentGroups`
- `groupKeys`：`Object.keys(currentGroups.value)` — 集采时返回 `['股份', '环境']`，集租时返回 `['股份']`
- `currentGroup`：`currentGroups.value?.[activeGroup.value] || {}`
- `currentSummary`：根据 tab 选择 `props.summary` 或 `props.rentSummary`，key 用 `activeGroup + (集采 ? '集采项目' : '集租项目')`

**修改 switchTab**：切换时重置 `activeGroup` 为 `'股份'`，再调用 `initChart()`

**修改 props**：新增 `rentSummary`（Object）、`rentGroups`（Object）

### 3. UnifiedDashboard.vue — 传入新 props

```vue
<MaterialProcurementChart
  :title="procurementData.title"
  :summary="procurementData.summary"
  :groups="procurementData.groups"
  :rent-summary="procurementData.rentSummary"
  :rent-groups="procurementData.rentGroups"
  :current-unit="selectedFilter"
/>
```

## 验证步骤
1. 默认"集采"Tab，显示股份/环境两个按钮，图表正常
2. 点击"集租"Tab，只显示"股份集租项目"按钮，横轴显示"塔吊、施工电梯"
3. 大标签显示集租汇总数据
4. 图表柱子+3条折线正常显示
