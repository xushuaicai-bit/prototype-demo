# 下钻报表自动传递筛选条件

## 问题分析

### 当前流程

```
UnifiedDashboard emit('navigate', { view:'revenue', report:'revenue-summary', filter:{ unit:'管网事业部' } })
  → App.handleNavigate() 只取 view+report 切换菜单，**丢弃了 filter**
  → RevenueReport :report-filter="{}"  ← 永远是空对象！
  → 报表收不到任何筛选条件
```

### 根因

[App.vue](src/App.vue) L23/L30/L37：所有报表的 `:report-filter` 硬编码为 `{}`，`handleNavigate()` 未保存下钻时的 filter。

***

## 修改方案

### 文件1：App.vue

**改动1** — 新增 `navigateFilter` ref 存储下钻筛选条件

```js
const navigateFilter = ref({})
```

**改动2** — `handleNavigate()` 保存 filter

```js
const handleNavigate = (data) => {
  // 保存下钻筛选条件（有则保存，无则清空）
  if (data && data.filter) {
    navigateFilter.value = data.filter
  } else {
    navigateFilter.value = {}
  }
  // ... 原有逻辑不变
}
```

**改动3** — `handleMenuChange()` 直接点击菜单时清空筛选

```js
const handleMenuChange = ({ menu, subMenu, key }) => {
  currentMenu.value = menu
  currentSubMenu.value = subMenu || ''
  navigateFilter.value = {}  // 直接点菜单不带筛选
}
```

**改动4** — 所有报表组件绑定 `navigateFilter`

```html
<RevenueReport :report-filter="navigateFilter" ... />
<SupplyChainReport :report-filter="navigateFilter" ... />
<MarketReport :report-filter="navigateFilter" ... />
```

### 效果对照

| 操作场景                               | navigateFilter            | 报表收到    |
| ---------------------------------- | ------------------------- | ------- |
| 筛选栏选"管网事业部" → 点击KPI卡片下钻            | `{ unit: '管网事业部' }`       | 自动选中该单位 |
| 筛选栏选"生态事业部" → 点击偏差单位下钻             | `{ unit: '环境建设' }`        | 自动选中该单位 |
| 筛选栏选"全部" → 下钻                      | `{ unit: '全部' }`          | 不限单位    |
| 直接侧边栏点击"营收汇总表"                     | `{}`                      | 无预设筛选   |
| 下钻时带 warningLevel/hasPlanRevenue 等 | `{ warningLevel: 'red' }` | 对应筛选项生效 |

### 注意事项

* UnifiedDashboard 中部分 emit 的 `filter` 是字符串（如 L598 `filter: selectedFilter.value`），需统一规范化为对象格式 `{ unit: xxx }`，或由 App.vue 兼容处理

* RevenueReport 已支持 `reportFilter.unit` 传给 RevenueSummary 的 `initial-filter`

* SupplyChainReport 当前未使用 `reportFilter` prop，但保留接口以便后续扩展
