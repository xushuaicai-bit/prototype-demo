# 预警等级去掉"已签订"调整计划

## 概述
用户要求：预警等级字段值只保留 **橙色预警、红色预警、黄色预警、未签订** 四种，去掉"已签订"。筛选条件同步调整。

## 当前状态

涉及3处代码：
1. **L325** `allSignStatus = ['已签订', '红色预警', '橙色预警', '黄色预警', '未签订']` — 下拉选项含"已签订"
2. **L1056-1063** `getWarningLabel()` — 第一行 `if (row.signStatus === '已签订') return '已签订'` 会返回"已签订"
3. **L1001** `tableData` computed 中筛选逻辑 — `contract.signStatus !== selectedSignStatus.value` 按原始 signStatus 值匹配，但 signStatus 只有 '已签订'/'未签订' 两种，与新的预警等级值（红色/橙色/黄色预警）无法正确匹配

## 修改步骤

### 步骤1：修改 allSignStatus 选项列表
**文件**: [SupplierContractReport.vue](src/components/supply-chain/SupplierContractReport.vue#L325)
- 从 `['已签订', '红色预警', '橙色预警', '黄色预警', '未签订']`
- 改为 `['红色预警', '橙色预警', '黄色预警', '未签订']`

### 步骤2：修改 getWarningLabel 函数
**文件**: [SupplierContractReport.vue](src/components/supply-chain/SupplierContractReport.vue#L1056-L1063)
- 删除 `if (row.signStatus === '已签订') return '已签订'` 分支
- 已签订的合同（overdueDays=0）会走到最后的 `return '未签订'`
- 最终逻辑：
```js
const getWarningLabel = (row) => {
  const days = row.overdueDays
  if (days >= 60) return '红色预警'
  if (days >= 45) return '橙色预警'
  if (days >= 21) return '黄色预警'
  return '未签订'
}
```

### 步骤3：修复筛选条件匹配逻辑
**文件**: [SupplierContractReport.vue](src/components/supply-chain/SupplierContractReport.vue#L1001)
- 当前：`if (selectedSignStatus.value && contract.signStatus !== selectedSignStatus.value) return`
- 问题：`contract.signStatus` 原始值为 `'已签订'` 或 `'未签订'`，与 `'红色预警'` 等新值永远不匹配
- 改为：用 `getWarningLabel(contract)` 做显示值匹配
```js
if (selectedSignStatus.value && getWarningLabel(contract) !== selectedSignStatus.value) return
```

## 验证方式
1. 表格预警等级列不再出现"已签订"，只有红/橙/黄预警和未签订四种
2. 筛选栏下拉框只有4个选项（无"已签订"）
3. 选择任意预警等级筛选项后表格能正确过滤
