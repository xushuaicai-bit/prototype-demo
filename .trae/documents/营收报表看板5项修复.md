# 营收报表/看板 5项修复计划

## Summary

修复营收报表和营收看板中的 5 个问题：明细表列标题动态化、汇总表筛选项隐藏、图表横坐标旋转、图表纵坐标空间、汇总表表头溢出。

## 问题逐一分析

---

### 问题 1 [中等] — 营收明细表"当月/上月"字段改为动态"xx月"

**文件**: [RevenueDetail.vue](src/components/revenue/RevenueDetail.vue)

**现状**: 表格中以下列标题是写死的静态文字：
- L277: `label="当月完成营收"`
- L310: `label="当月上报股份营收"`  
- L343: `label="截止当月剩余合同存量"`

搜索栏已有年月筛选器（L97-106）：`filters.detailYear` + `filters.detailMonth`

**需求**: 列标题应随搜索栏选中的年月动态变化。例如用户选择 **2026年5月** → 标题变为 **"5月完成营收"**、**"5月上报股份营收"**、**"截止5月剩余合同存量"**

**方案**: 新增 computed 属性 `dynamicColumnLabels`，根据 `filters.detailMonth` 动态生成标签文本（未选月份时保持原"当月"文案）。模板中将 3 处硬编码 label 改为引用 computed 值。

```js
// 新增 computed
const dynamicMonthLabel = computed(() => {
  const m = filters.value.detailMonth
  return m ? `${m}月` : '当月'
})
```

模板改动示例：
```html
<!-- 修改前 -->
<label="当月完成营收">
<!-- 修改后 -->
:label="dynamicMonthLabel + '完成营收'"
```

同步修改其余 2 处。

### 问题 2 [简单] — 营收汇总表施工类的年月筛选项去掉

**文件**: [RevenueSummary.vue](src/components/revenue/RevenueSummary.vue)

**现状**: L52-62 的年月筛选器（summaryYear + summaryMonth）在**所有 Tab** 下都显示。

**需求**: 仅在"施工类总营收"Tab 时**隐藏**该筛选项（其他 Tab 保留）。

**方案**: 将年月筛选器外层包裹 `v-show` 或 `v-if`，条件为 `activeTab !== 'construction'`：

```html
<!-- 修改前 -->
<div class="flex items-center">
  <label>年月：</label>
  ...
</div>

<!-- 修改后 -->
<div v-show="activeTab !== 'construction'" class="flex items-center">
  <label>年月：</label>
  ...
</div>
```

使用 `v-show` 而非 `v-if`，避免切换 Tab 时筛选值丢失。

### 问题 3 [简单] — 营收月度统计横坐标斜着显示

**涉及文件**:
- [RevenueDashboard.vue#L710-L711](src/views/RevenueDashboard.vue#L710-L711)
- [UnifiedDashboard.vue#L936-L937](src/views/UnifiedDashboard.vue#L936-L937)

**现状**: 营收月度统计图表的 xAxis 配置为：
```js
xAxis: { type: 'category', data: ['2025-11', '2025-12', '2026-01', '2026-02', '2026-03', '2026-04'] }
```
**没有设置 axisLabel**，横坐标文字水平显示，在有限宽度内容易重叠吞字。

**方案**: 给两处 xAxis 都加上 axisLabel 配置，旋转 30-45 度并设置 interval: 0：

```js
xAxis: {
  type: 'category',
  data: [...],
  axisLabel: {
    interval: 0,
    rotate: 35,
    fontSize: 11,
    margin: 12
  }
}
```

与右侧"当年累计完成营收情况"图表的 axisLabel 风格统一（后者已有 rotate:35）。

### 问题 4 [简单] — 当年累计完成营收情况图表下移

**涉及文件**:
- [RevenueDashboard.vue#L753](src/views/RevenueDashboard.vue#L753)
- [UnifiedDashboard.vue#L979](src/views/UnifiedDashboard.vue#L979)

**现状**: grid 配置为 `{ left: '10%', right: '10%', bottom: '24%', top: '5%', containLabel: true }`
top 仅 5%，导致纵坐标轴名称（如"累计营收(万元)"、"项目数量"）顶部被裁剪无法完整显示。

**方案**: 将 `grid.top` 从 `'5%'` 调整为 `'12%'` 或 `'15%'`，给纵坐标留出足够空间：

```js
grid: { left: '10%', right: '10%', bottom: '24%', top: '14%', containLabel: true }
```

### 问题 5 [中等] — 营收汇总表施工类表头最后几列被吞掉

**文件**: [RevenueSummary.vue#L87-L301](src/components/revenue/RevenueSummary.vue#L87-L301)

**现状**: 施工类总营收表格列非常多（约 18+ 列），虽然外层有 `overflow-x-auto`（L87），但表格本身可能因容器宽度限制或 min-width 设置不当导致最后几列（如"截止本月剩余合同存量"、"下月计划"）的表头不可见。

**方案**: 
1. 确认 el-table 容器的 `style="max-width: 100%"` 不限制子元素溢出滚动
2. 给 el-table 添加 `min-width` 确保所有列有足够空间（参考合计行已设置的 `min-width: 2500px`）
3. 检查是否有父级容器的 `overflow: hidden` 导致截断

具体改动：将 L87 的 `<div>` 上增加明确的 min-width 或调整 table 属性确保横向滚动生效。
