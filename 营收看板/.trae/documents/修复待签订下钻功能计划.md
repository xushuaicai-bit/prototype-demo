# 修复待签订下钻功能计划

## 问题分析
当前供应链看板中，只有"已签订"数字支持点击下钻，但"待签订"数字没有绑定点击事件，无法下钻。

## 解决方案

### 修改文件
`src/components/dashboard/SupplyChainTimeliness.vue`

### 具体修改
1. **给"待签订"数字添加点击事件**
   - 在第131行，给待签订数字添加点击事件处理
   - 添加与"已签订"相同的样式（蓝色字体、hover下划线效果）

2. **添加处理函数**
   - 添加 `handlePendingClick` 函数
   - 下钻时筛选：对应合同分类 + 签订状态=未签订

### 修改内容
在第130-132行，把待签订数字改为可点击的：
```html
<div class="text-gray-500">
  <span class="text-gray-400 font-medium">○</span> 待签订 
  <span 
    class="ml-1 font-medium text-gray-600 cursor-pointer hover:underline"
    @click="handlePendingClick(item.name)"
  >{{ item.circle.pending }}</span>
</div>
```

添加处理函数：
```javascript
const handlePendingClick = (categoryName) => {
  const category = categoryMap[categoryName]
  if (category) emit('drill-down', { category, signStatus: '未签订' })
}
```

## 验证要点
- 点击待签订数字可以下钻到报表
- 筛选条件正确：对应分类 + 签订状态=未签订
- 样式与已签订保持一致
