# 供应链看板进度环优化计划

## 📋 需求分析

1. **圆形进度环显示内容调整**：
   - 进度环中心显示：应签订总数（数量）
   - 悬浮显示：百分比（已签订/应签订）
   
2. **已签订和应签订支持下钻**：
   - 点击"已签订"数字 → 下钻到报表，筛选该分类且签订状态=已签订
   - 点击"应签订"数字（通过进度环或标题）→ 下钻到报表，筛选该分类

3. **数据一致性**：
   - 确保筛选出的数据条目数与看板显示一致

## 📁 文件修改清单

| 文件路径 | 修改内容 | 优先级 |
|---------|---------|-------|
| `src/data/mockData.js` | 更新 subCircles 数据结构 | 高 |
| `src/components/dashboard/SupplyChainTimeliness.vue` | 修改进度环显示、悬浮提示、下钻逻辑 | 高 |

## 📝 具体实施步骤

### 步骤1：更新 mockData.js 数据结构

修改 `supplyChainTimelinessData.subCircles` 数据，增加：
- `total`: 应签订总数（目前用 signed + pending）
- 确保数据完整性：signed + pending = total

```javascript
subCircles: [
  { 
    percentage: 55, 
    signed: 55, 
    pending: 50, 
    total: 105  // 新增：应签订总数
  },
  // ... 其他分类
]
```

### 步骤2：修改 SupplyChainTimeliness.vue 进度环显示

**进度环中心显示调整**：
- 第一行：应签订总数（如 "105"）
- 第二行："应签订"
- 第三行："总数"

**悬浮提示（Tooltip）**：
- 显示："完成率：XX%"
- 计算公式：percentage = (signed / total) * 100

### 步骤3：为已签订和应签订添加下钻功能

**已签订数字点击**：
- 点击"已签订 55"中的数字部分
- 下钻到报表，筛选：contractCategory + signStatus = 已签订

**应签订点击**（保持现有）：
- 点击进度环或分类标题
- 下钻到报表，筛选：contractCategory

### 步骤4：添加工具提示组件

使用 Element Plus 的 `el-tooltip` 组件包裹进度环
