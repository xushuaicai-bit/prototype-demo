# 界面整体架构调整计划

## 一、现状分析

### 当前架构

* **顶部导航栏 (Header.vue)**: 管理总览、市场管理、产运管理、经营管理、安全管理、资金管理

* **左侧菜单栏 (Sidebar.vue)**: 固定显示所有菜单项

  * 供应链管理看板

  * 生产看板

  * 营收管理看板

  * 技术科研看板

  * 市场管理看板

  * 营收报表（含子菜单）

  * 供应链报表（含子菜单）

  * 市场报表（含子菜单）

### 问题

* 顶部导航和左侧菜单没有关联

* 所有菜单项平铺显示，缺乏层次结构

* 不符合业务模块化管理的要求

### 步骤 3: 修改 App.vue

**文件**: `src/App.vue`

修改内容：

1. **添加 nav 状态管理**:

   ```javascript
   const currentNav = ref('市场管理')
   ```

2. **传递 nav 给 Header 和 Sidebar**:

   ```html
   <Sidebar 
     :current-nav="currentNav" 
     @menu-change="handleMenuChange" 
   />
   <Header @nav-change="handleNavChange" />
   ```

3. **处理导航切换逻辑**:

   ```javascript
   const handleNavChange = (navName) => {
     currentNav.value = navName
     // 切换导航时重置菜单状态
     currentMenu.value = ''
     currentSubMenu.value = ''
     
     // 如果该导航下有默认菜单，自动选择第一个
     const menus = getMenuForNav(navName)
     if (menus.length > 0) {
       // 自动选择第一个菜单项
       handleFirstMenuSelection(menus[0])
     }
   }
   ```

4. **更新 viewMap 以支持新结构**:
