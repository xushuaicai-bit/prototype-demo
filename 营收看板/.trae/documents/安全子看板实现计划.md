# 安全子看板实现计划

## 一、需求分析

### 1.1 需求概述
在安全管理模块中，左侧新增"安全子看板"菜单，实现两个Tab切换界面：
- **安全在线**：展示实名制、隐患排查、视频监控等在线安全数据
- **过程管理**：展示安全检查完成情况、排名、趋势图等过程管理数据

### 1.2 界面还原内容

#### 安全在线界面
- 顶部Tab导航：安全在线 / 过程管理（仅这两个切换）
- 顶部统计卡片行：8个指标卡片（生产项目、安全的管项目、开通实名制项目、未及时登记实名制项目、开通视频监控项目、开通AI监控项目、近一周开展隐患排查项目、重大安全风险项目）
- 主体三列布局：
  - 实名制板块：登记人数、安全质量人数、安全员未到项目、预警明细
  - 隐患排查板块：排查单数、整改率、隐患项目统计、预警项目
  - 视频监控板块：监控数、AI点位、在线率、隐患项目、未安装项目
- 右侧预警项面板：重置隐患、隐患整改不及时、重大事故隐患、上级单位检查、安全投入预警
- 底部：重大安全风险项目卡片列表 + 项目现场AI抓拍

#### 过程管理界面
- 顶部单位Tab：总体情况、上海域浦、市政集团、上海路桥、城市环境、上海能建、畅凝物资
- 左上：总体情况表格（单位、项目数、本周未完成日历、已完成/未完成数量）
- 左下：未完成/已完成项目列表（筛选 + 数据表格 + 分页）
- 右上：排名列表（各单位完成率排名）
- 右中：完成情况趋势图（近30天日均完成数量）
- 右下：访问量统计（访问次数/人数趋势图）

## 五、潜在依赖和注意事项

1. **ECharts图表**：过程管理中有趋势图和访问量统计图，需使用ECharts，项目已安装 echarts ^5.5.0
2. **Element Plus**：表格、分页、按钮等可使用Element Plus组件，项目已安装
3. **Mock数据**：所有数据使用静态mock数据，无需对接后端
4. **样式一致性**：严格遵循现有看板的视觉风格（蓝色主题、白色卡片、圆角xl、shadow-sm）
5. **不改动其他模块**：仅修改安全管理相关内容，不影响其他模块

## 六、风险处理

1. **界面细节不明确**：以截图为主要还原依据，文字内容从截图中提取，数值使用合理的mock数据
2. **交互细节**：hover效果、点击切换等基础交互全部实现，复杂的业务交互（如下钻）暂不实现
3. **图表数据**：趋势图数据生成合理的模拟数据，确保图表展示效果美观
