# 供应链报表实现计划

## 需求分析

用户要求在营收报表目录下创建新的供应链报表子目录，并创建第一个报表"供应商合同签订报表"。

### 报表数据展示要求

* 采用树形结构展示"项目"和"合同分类"两个层级维度

* 实现一个项目对应多个合同类型的层级展示

* 每个合同类型作为独立行显示其相关数据

### 筛选条件配置

* 项目名称筛选器（单选）

* 基层单位筛选器（单选）

* 项目状态筛选器（单选）

* 合同分类筛选器（单选）

* 支持取消选项和实时筛选

## 文件结构规划

```
src/
├── components/
│   └── supply-chain/           # 新增供应链报表组件目录
│       └── SupplierContractReport.vue  # 供应商合同签订报表组件
├── views/
│   └── SupplyChainReport.vue   # 新增供应链报表视图
├── App.vue                     # 修改：添加供应链报表视图路由
└── components/
    └── layout/
        └── Sidebar.vue         # 修改：添加供应链报表菜单
```

## 实施步骤

### 步骤1：创建供应链报表组件目录和报表组件

* 创建 `src/components/supply-chain/SupplierContractReport.vue`

* 实现树形结构表格展示

* 实现四个筛选器（项目名称、基层单位、项目状态、合同分类）

* 添加模拟数据

### 步骤2：创建供应链报表视图

* 创建 `src/views/SupplyChainReport.vue`

* 引入并展示供应商合同签订报表组件

### 步骤3：修改侧边栏菜单

* 修改 `src/components/layout/Sidebar.vue`

* 添加"供应链报表"菜单项及子菜单

### 步骤4：修改主应用组件

* 修改 `src/App.vue`

* 添加供应链报表视图路由逻辑

## 技术实现要点

1. **树形结构展示**：使用 Element Plus 的 el-table 组件，通过 row-span 实现项目名称合并
2. **筛选器实现**：使用 Element Plus 的 el-select 组件，支持单选和清空功能
3. **实时筛选**：使用 Vue 的 computed 属性实现数据过滤
4. **样式保持一致**：参考现有营收报表的蓝色主题样式
