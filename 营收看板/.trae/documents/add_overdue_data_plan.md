# 供应链报表增加超期模拟数据计划

## 需求分析

当前模拟数据中的超期天数（overdueDays）都在 0-7 天范围内，无法测试超期筛选功能。需要添加更多数据，覆盖三个超期区间：
- 超期21~44天
- 超期45~60天
- 超期60天及以上

## 修改文件

- `src/components/supply-chain/SupplierContractReport.vue`

## 实施步骤

### 步骤1：修改现有项目的合同数据
为现有项目添加更多合同，包含各超期区间的数据：

**P001 城乡水务局供水管网改造工程** 新增合同：
- 专业分包：超期30天（21~44天区间）
- 劳务分包：超期55天（45~60天区间）

**P002 生态环境治理项目** 新增合同：
- 专业分包：超期65天（60天以上区间）
- 劳务分包：超期25天（21~44天区间）

**P004 市政道路新建项目** 修改：
- 专业分包：超期35天（21~44天区间）
- 新增劳务分包：超期70天（60天以上区间）

### 步骤2：新增项目数据
新增2个项目，包含更多超期合同：

**P006 管道改造二期工程**（管道工程）：
- 专业分包：超期22天
- 劳务分包：超期48天
- 材料/设备采购：超期75天

**P007 水务综合管理平台**（管道分公司）：
- 专业分包：超期40天
- 劳务分包：超期58天
- 周转材料/设备租赁：超期80天

### 步骤3：更新项目名称筛选选项
在 `allProjects` 数组中添加新项目名称

## 预期数据分布

| 超期区间 | 数据条数 |
|---------|---------|
| 0-20天 | 多条（正常数据） |
| 21~44天 | 4条以上 |
| 45~60天 | 3条以上 |
| 60天以上 | 3条以上 |

## 预期效果

1. 选择"超期21~44天"时，能看到多条数据
2. 选择"超期45~60天"时，能看到多条数据
3. 选择"超期60天及以上"时，能看到多条数据
4. 各筛选区间都有足够的数据进行测试
