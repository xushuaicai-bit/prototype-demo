# 市场跟踪管理页面实现计划

## 需求分析

用户要求还原一个独立的"市场跟踪管理"页面，特点：
- 页面名称：市场跟踪管理
- 位置：市场报表下
- 无菜单设计（独立页面）
- 参考URL：`https://www.smart-worksite.com/hj/#/micro/market/biz/ent/market/project/potential`

## 现有代码结构分析

```
src/
├── components/
│   └── market/
│       ├── ProjectTrackingReport.vue  # 项目跟踪数据表组件
│       ├── KeyProjectListReport.vue
│       ├── BidSummaryReport.vue
│       ├── WinningBidReport.vue
│       └── InBidReport.vue
├── views/
│   ├── MarketDashboard.vue    # 市场管理看板
│   └── MarketReport.vue       # 市场报表容器
└── data/
    └── mockData.js            # 模拟数据
```

## 实现方案

### 1. 创建独立视图页面

创建 `src/views/MarketTracking.vue`，作为独立的市场跟踪管理页面，不包含侧边栏和顶部导航。

### 2. 页面结构设计

```
┌─────────────────────────────────────────────────────────────┐
│  市场跟踪管理                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 筛选区: 公司选择 | 年份选择 | 项目类型筛选           │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 项目跟踪数据表                                       │   │
│  │ 分公司 | AAA | AA | A | 小计                        │   │
│  │ ...数据行...                                         │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 底部操作: 导出Excel                                  │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 3. 文件修改清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/views/MarketTracking.vue` | 新建 | 独立的市场跟踪管理页面 |
| `src/App.vue` | 修改 | 添加市场跟踪管理页面路由 |

### 4. 数据来源

使用现有 `ProjectTrackingReport.vue` 中的模拟数据结构，包含：
- 分公司名称
- AAA/AA/A等级项目数量和金额
- 小计汇总

### 5. 技术要点

- 使用 Element Plus 表格组件展示数据
- 使用 Tailwind CSS 进行样式设计
- 响应式布局适配不同屏幕
- 保留导出Excel功能

## 风险评估

| 风险 | 等级 | 处理方案 |
|------|------|----------|
| 样式冲突 | 低 | 使用独立的样式作用域 |
| 数据不一致 | 低 | 使用统一的mock数据 |
| 路由冲突 | 低 | 使用唯一的路由标识 |

## 执行步骤

1. 创建 `MarketTracking.vue` 视图组件
2. 修改 `App.vue` 添加路由
3. 测试构建和运行
