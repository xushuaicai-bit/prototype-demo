# 安全子看板嵌入外部系统实现计划

## 一、需求概述

将安全子看板的内容从自建的三个Tab（安全在线、过程管理、实名制专版）替换为通过 iframe 嵌入外部系统：
- 外部URL：`https://szsg-test.stec.net/#/enterprise/statistics/safety/board?t=1783930391415`
- 登录方式：iframe + 首次手动登录（浏览器缓存 cookie，后续自动免登录）
- 隐藏自建的三个Tab和组件渲染，但保留组件文件不删除

## 二、代码调研

### 当前状态
- [SafetyDashboard.vue](file:///e:/trae_demo_env/营收看板/src/views/SafetyDashboard.vue) 当前包含3个Tab切换 + 3个子组件渲染
- 三个子组件文件：SafetyOnline.vue、ProcessManagement.vue、RealNameSystem.vue（保留不删除）
- SafetyDashboard 通过 [App.vue](file:///e:/trae_demo_env/营收看板/src/App.vue#L41-L47) 的 `<component :is="currentViewComponent">` 渲染
- 主内容区：`<main class="flex-1 overflow-auto p-4">`，父容器 `overflow-hidden`，高度受限

### 外部系统
- 数智化项目现场管理平台（SPA 应用）
- 首次打开需手动登录，之后浏览器缓存 session cookie 实现免登录

## 三、修改方案

### 修改文件：SafetyDashboard.vue

将整个组件内容替换为 iframe 嵌入：

1. **模板**：移除 Tab 导航和子组件渲染，替换为全屏 iframe
   - iframe 使用 `w-full h-full border-0` 填满内容区
   - 添加加载中提示（loading 状态）
   
2. **脚本**：
   - 移除 SafetyOnline、ProcessManagement、RealNameSystem 的导入
   - 移除 activeTab 和 topTabs
   - 保留 iframe 的 URL 配置

3. **样式**：
   - 根容器使用 `h-full` 填满父级
   - iframe 圆角无边框

### 不改动的文件
- SafetyOnline.vue、ProcessManagement.vue、RealNameSystem.vue（保留文件，仅不渲染）
- Sidebar.vue、App.vue（菜单和路由无需改动）
- chinaMap.js（工具文件保留）

## 四、实现细节

```vue
<template>
  <div class="h-full relative">
    <div v-if="loading" class="absolute inset-0 flex items-center justify-center bg-white rounded-xl">
      <div class="text-center">
        <div class="animate-spin w-10 h-10 border-4 border-blue-500 border-t-transparent rounded-full mx-auto mb-3"></div>
        <p class="text-gray-500 text-sm">正在加载安全在线管控平台...</p>
      </div>
    </div>
    <iframe
      :src="boardUrl"
      class="w-full h-full border-0 rounded-xl bg-white"
      allowfullscreen
      @load="loading = false"
    />
  </div>
</template>
```

## 五、风险处理

1. **X-Frame-Options 限制**：外部系统可能设置 `X-Frame-Options: DENY` 导致 iframe 无法加载。如遇到此问题，需在外部系统服务器配置允许跨域嵌入
2. **Cookie 跨域**：iframe 中的 cookie 可能受 SameSite 策略影响。如首次登录后仍无法缓存，可能需要用户在浏览器设置中允许第三方 cookie
3. **session 过期**：cookie 过期后需重新手动登录，属预期行为
