# 供应链报表新增合同日期字段和预警功能计划

## 需求分析

在供应商合同签订报表中，在超期天数字段前新增3个字段：

### 新增字段

1. **计划签订日期** - 每行都有值
2. **实际签订日期** - 已签订有值，未签订无值
3. **签订状态** - 已签订、未签订

### 预警规则

* 未签订且超期21\~44天：黄色预警

* 未签订且超期45\~60天：橙色预警

* 未签订且超期60天及以上：红色预警

### 其他需求

* 新增10行模拟数据

## 修改文件

* `src/components/supply-chain/SupplierContractReport.vue`

## 实施步骤

### 步骤1：修改表格列结构

在超期天数字段前添加3个新列：

```
计划签订日期 → 实际签订日期 → 签订状态 → 超期天数
```

### 步骤2：更新模拟数据

为每个合同添加新字段：

* `plannedDate` - 计划签订日期（如 '2026-01-15'）

* `actualDate` - 实际签订日期（已签订有值，未签订为 '' 或 null）

* `signStatus` - 签订状态（'已签订' 或 '未签订'）

### 步骤3：更新现有数据

为现有的合同数据添加新字段值

### 步骤4：新增10行数据

创建更多合同记录，覆盖不同的超期区间和签订状态

### 步骤5：实现预警样式

添加预警显示逻辑：

* 根据签订状态和超期天数，给签订状态列添加颜色样式

* 黄色预警：`bg-yellow-100 text-yellow-800`

* 橙色预警：`bg-orange-100 text-orange-800`

* 红色预警：`bg-red-100 text-red-800`

* 正常状态：默认样式

### 步骤6：更新项目名称筛选选项

添加新项目名称（如果有）

## 数据结构示例


## 预警规则总结

| 签订状态 | 超期天数    | 预警等级 | 样式   |
| ---- | ------- | ---- | ---- |
| 已签订  | 任意      | 无    | 默认   |
| 未签订  | 0-20天   | 无    | 默认   |
| 未签订  | 21\~44天 | 黄色   | 黄色背景 |
| 未签订  | 45\~60天 | 橙色   | 橙色背景 |
| 未签订  | 60天及以上  | 红色   | 红色背景 |

2. 已签订的显示实际签订日期，未签订的显示空
3. 签订状态列根据条件显示不同颜色预警
4. 新增10行数据后，共有更多数据进行测试
