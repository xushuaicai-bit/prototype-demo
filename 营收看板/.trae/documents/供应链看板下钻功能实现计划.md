# 供应链看板下钻功能实现计划

## 一、实现目标

实现从供应链看板（下游供应商合同签订及时性模块）点击各统计指标，下钻到供应商合同签订报表并自动预填筛选条件的功能。

## 二、文件修改清单

| 文件路径 | 修改内容 | 优先级 |
|---------|---------|-------|
| `src/components/dashboard/SupplyChainTimeliness.vue` | 添加点击事件，处理下钻逻辑 | 高 |
| `src/App.vue` | 更新路由跳转逻辑，支持传递筛选参数 | 高 |
| `src/views/SupplyChainReport.vue` | 接收并传递筛选参数给报表组件 | 高 |
| `src/components/supply-chain/SupplierContractReport.vue` | 支持从外部设置筛选条件 | 高 |

## 三、具体实现步骤

### 3.1 修改 SupplyChainTimeliness.vue

**目标**: 为各个可点击元素添加点击事件，触发下钻

**具体修改**:
1. 顶部6个统计卡片（应签订总数、待签订总数、已签订总数、黄色预警、橙色预警、红色预警）添加点击事件
2. 4个合同分类模块添加点击事件：
   - 分类标题点击
   - 黄色/橙色/红色预警数字点击
   - 进度环区域点击
3. 添加 `handleClick` 方法，构建筛选参数对象并触发事件
4. 定义 `defineEmits` 发送 `drill-down` 事件

**筛选参数结构**:

## 四、技术实现细节

### 4.1 点击样式反馈
- 可点击元素添加 `cursor: pointer` 样式
- 鼠标悬停时添加轻微的背景色变化
- 点击时添加短暂的视觉反馈

### 4.2 事件处理流程
```
SupplyChainTimeliness.vue (用户点击)
    ↓
drill-down 事件
    ↓
SupplyChainDashboard.vue
    ↓
App.vue (更新路由状态)
    ↓
SupplyChainReport.vue
    ↓
SupplierContractReport.vue (应用筛选)
```

### 4.3 状态管理考虑
选项1: 通过 props 层层传递（简单，适合当前架构）
选项2: 使用 Pinia 管理全局状态（更灵活，可扩展性好）
选项3: 使用 sessionStorage 持久化（支持刷新后保留）

**建议采用选项1 + 选项3的组合**
