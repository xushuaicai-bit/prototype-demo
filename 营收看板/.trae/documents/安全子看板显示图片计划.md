# 安全子看板显示图片实现计划

## 一、需求概述

将安全子看板从 iframe 链接外部系统改为直接显示安全在线管控平台界面的图片。用户提供了目标界面截图，需要在安全子看板中直接展示该图片。

## 二、代码调研结论

### 当前状态
- [SafetyDashboard.vue](file:///e:/trae_demo_env/营收看板/src/views/SafetyDashboard.vue) 当前使用 iframe 嵌入外部 URL
- 通过 [externalUrls.js](file:///e:/trae_demo_env/营收看板/src/config/externalUrls.js) 的 safetyBoard 配置获取 URL

### 目标变更
- 移除 iframe 逻辑，直接显示图片
- 使用图片生成 API 根据截图内容生成图片
- 图片应自适应容器大小

## 三、需要修改的文件

### 修改文件
1. **SafetyDashboard.vue** - 移除 iframe，替换为图片展示

### 不改动文件
- externalUrls.js（保留配置，不删除）
- 其他组件文件（SafetyOnline.vue、ProcessManagement.vue、RealNameSystem.vue）

## 四、实现步骤

### 步骤1：修改 SafetyDashboard.vue
- 移除 iframe 和 loading 状态
- 添加 `<img>` 标签显示安全在线管控平台图片
- 使用图片生成 API：`https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image`
- 图片尺寸使用 `landscape_16_9`（适合仪表板界面）
- 添加自适应样式（`w-full h-full object-contain`）

### 步骤2：验证效果
- 启动开发服务器，查看安全子看板显示效果
- 确认图片正常加载且布局适配

## 五、潜在依赖和注意事项

1. **图片生成 API**：使用 Trae 内置的文本转图片 API，需要构建准确的 prompt 来还原截图中的安全在线管控平台界面
2. **图片尺寸**：使用 landscape_16_9 尺寸，配合 `object-contain` 确保图片完整显示
3. **网络依赖**：图片生成需要网络请求，首次加载可能需要几秒

## 六、风险处理

1. **图片生成失败**：添加加载状态提示和错误处理
2. **图片内容差异**：prompt 描述尽量详细，确保生成的图片接近用户提供的截图
3. **布局适配**：使用 `object-contain` 避免图片拉伸变形
