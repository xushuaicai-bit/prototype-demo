# 供应链看板与供应商合同签订报表对应关系分析

## 一、总体架构

### 1.1 组件结构
- **供应链管理看板**: `SupplyChainDashboard.vue`
- **下游供应商合同签订及时性模块**: `SupplyChainTimeliness.vue`
- **供应商合同签订报表**: `SupplierContractReport.vue`
- **报表容器**: `SupplyChainReport.vue`

### 1.2 数据流向
```
mockData.js (数据源)
    ↓
SupplyChainDashboard.vue (看板展示)
    ↓ (点击下钻)
SupplyChainReport.vue (报表容器)
    ↓
SupplierContractReport.vue (报表详情)
```

## 二、顶层统计卡片对应关系

### 2.1 应签订总数 → 报表筛选
| 看板元素 | 数据值 | 报表筛选条件 | 对应字段 |
|---------|--------|-------------|---------|
| 应签订总数 | 4622 | 全部项目 | - |
| 待签订总数 | 1222 | 签订状态 = "未签订" | `signStatus` |
| 已签订总数 | 3500 | 签订状态 = "已签订" | `signStatus` |

**下钻逻辑**:
- 点击"应签订总数" → 进入报表，不设置筛选，显示全部合同
- 点击"待签订总数" → 进入报表，自动筛选"签订状态 = 未签订"
- 点击"已签订总数" → 进入报表，自动筛选"签订状态 = 已签订"

## 三、合同分类模块对应关系

### 3.1 合同分类清单
| 合同分类名称 | 预警指标 | 进度环数据 | 对应筛选 |
|-------------|---------|-----------|---------|
| 专业分包合同签订及时性 | 黄:10, 橙:0, 红:10 | 95% / 已签55 / 待签50 | `category = "专业分包"` |
| 劳务分包合同签订及时性 | 黄:0, 橙:0, 红:10 | 95% / 已签55 / 待签50 | `category = "劳务分包"` |
| 周转材料/设备租赁合同签订及时性 | 黄:0, 橙:0, 红:10 | 95% / 已签55 / 待签50 | `category = "周转材料/设备租赁"` |
| 材料/设备采购合同签订及时性 | 黄:0, 橙:0, 红:10 | 95% / 已签55 / 待签50 | `category = "材料/设备采购"` |

### 3.2 点击下钻行为
**合同分类标题点击**:
- 点击"专业分包合同签订及时性" → 进入报表，自动筛选"合同分类 = 专业分包"
- 点击"劳务分包合同签订及时性" → 进入报表，自动筛选"合同分类 = 劳务分包"
- 点击"周转材料/设备租赁合同签订及时性" → 进入报表，自动筛选"合同分类 = 周转材料/设备租赁"
- 点击"材料/设备采购合同签订及时性" → 进入报表，自动筛选"合同分类 = 材料/设备采购"

**预警数字点击**:
- 点击黄色预警数字 → 进入报表，筛选"合同分类 + 超期天数21~44天"
- 点击橙色预警数字 → 进入报表，筛选"合同分类 + 超期天数45~60天"
- 点击红色预警数字 → 进入报表，筛选"合同分类 + 超期天数60天及以上"

**进度环点击**:
- 点击进度环 → 进入报表，筛选该合同分类 + 可选择性筛选签订状态

## 四、报表现有筛选器

### 4.1 筛选器列表
| 筛选器名称 | 对应字段 | 可选值 |
|-----------|---------|-------|
| 项目名称 | `projectName` | 9个固定项目 |
| 基层单位 | `unit` | 9个事业部 |
| 项目状态 | `status` | 特建/在建/停工/完工/当年竣工 |
| 合同分类 | `category` | 专业分包/劳务分包/材料/设备采购/周转材料/设备租赁 |
| 超期天数 | `overdueDays` | 21~44天/45~60天/60天及以上 |

### 4.2 报表表格字段
| 字段名 | 说明 | 看板关联 |
|-------|------|---------|
| 项目编号 | `projectCode` | - |
| 项目名称 | `projectName` | - |
| 基层单位 | `unit` | - |
| 项目状态 | `status` | - |
| 合同分类 | `category` | ✓ |
| 分包合同名称 | `contractName` | - |
| 工作范围及内容 | `workScope` | - |
| 主要工程量 | `workVolume` | - |
| 初始策划金额 | `plannedAmount` | - |
| 合同选用形式 | `contractForm` | - |
| 计划签订日期 | `plannedDate` | - |
| 实际签订日期 | `actualDate` | - |
| 签订状态 | `signStatus` | ✓ |
| 超期天数 | `overdueDays` | ✓ |

## 六、实现要点

### 6.1 需要实现的功能
1. **点击事件处理**: 在 SupplyChainTimeliness 组件中为各元素添加点击事件
2. **参数传递**: 通过 App.vue 的 handleReportNavigate 传递筛选参数
3. **预填充筛选器**: SupplierContractReport 接收并自动设置筛选条件
4. **URL或状态管理**: 记录跳转来源，支持返回功能

### 6.2 待确认事项
1. 预警数字与超期天数的映射关系确认
2. 进度环数据与实际合同数的对应关系
3. 是否需要双向跳转（从报表返回看板）
4. 是否需要保留筛选记忆功能
