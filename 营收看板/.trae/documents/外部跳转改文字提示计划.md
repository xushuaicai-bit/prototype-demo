# 外部跳转改为文字提示计划

## 一、总结

将原来跳转到 `https://www.smart-worksite.com/hj/#` 的所有界面，从"静默无操作"改为显示文字提示"跳转至环境项管系统对应模块"。

当前 Demo 模式下（外部 URL 基础地址为空），点击下钻元素和 iframe 占位页面的行为需要调整：
- **点击下钻**（window.open 调用）：当前静默无操作 → 改为弹出 `ElMessage` 提示
- **iframe 占位**（EconomyDashboard）：当前显示"演示环境：外部系统未配置" → 改为显示"跳转至环境项管系统对应模块"

## 三、拟议变更

### 3.1 修改 `src/config/externalUrls.js`

**目的**：`openExternal()` 在 URL 为空时弹出提示，而非静默返回。

```js
// 新增 import
import { ElMessage } from 'element-plus'

// 修改 openExternal 函数
export function openExternal(url) {
  if (!url) {
    ElMessage.info('跳转至环境项管系统对应模块')
    return
  }
  window.open(url, '_blank')
}
```

### 3.2 修改 `src/components/dashboard/ProjectStatCard.vue`

**目的**：让 ProjectStatCard 也走 `openExternal()`，Demo 模式下点击时显示提示。

当前代码：
```js
const handleClick = () => {
  if (props.drillable && props.link) {
    window.open(props.link, '_blank')
  }
}
```

改为：
```js
import { openExternal } from '@/config/externalUrls'

const handleClick = () => {
  if (props.drillable) {
    openExternal(props.link)
  }
}
```

- 移除 `&& props.link` 条件，改为只要 `drillable` 就调用 `openExternal`
- `openExternal('')` 会触发 ElMessage 提示

### 3.3 修改 `src/views/EconomyDashboard.vue`

**目的**：iframe 占位提示文字改为"跳转至环境项管系统对应模块"。

当前（第 29-30 行）：
```html
<h3 class="text-lg font-medium text-gray-800 mb-2">演示环境</h3>
<p class="text-gray-600">经济管理看板外部系统未配置</p>
```

改为：
```html
<h3 class="text-lg font-medium text-gray-800">跳转至环境项管系统对应模块</h3>
```

## 四、假设与决策

1. **SafetyDashboard 不修改**：其原始 URL 为 `szsg-test.stec.net`（非 smart-worksite.com），不在用户指定范围
2. **ElMessage 类型用 `info`**：文字"跳转至环境项管系统对应模块"为信息性提示，用 `ElMessage.info()` 最合适
3. **ProjectStatCard 统一走 openExternal**：消除直接 `window.open` 调用，确保所有跳转入口行为一致
4. **iframe 占位保留锁图标**：仅改文字，不改 SVG 图标（用户未要求改图标）
